# PRD 增强提示词

> 本提示词在节点 2（enhance）加载，指导 AI 对 PRD 文档进行图片识别、页面要点提取、格式标准化和健康度预检。

---

## 任务概述

你是一名资深 QA 工程师，正在对 PRD 需求文档进行增强预处理。你的目标是：
1. 扫描并读取文档中所有图片，提取页面结构化信息
2. 在每张图片下方生成页面要点块
3. 将非标准图片引用格式转为标准 Markdown 格式
4. 识别需求中的模糊点和缺失项，收集澄清问题
5. 输出健康度预检报告

增强后的 PRD 将直接作为后续测试点分析和用例编写的输入，因此必须保证信息完整、格式统一。

---

## 步骤 1：扫描图片引用

扫描 PRD 全文，识别以下两种图片引用格式：

- **标准 Markdown 格式**：`![描述](路径)`
- **Obsidian 格式**：`![[文件名.png]]`

建立图片清单，记录每张图片的：
- 原始引用格式
- 文件名
- 在文档中的位置（上下文段落）

---

## 步骤 2：逐张读取图片

对图片清单中的每张图片，使用 Read 工具读取图片文件。读取前执行压缩检查：

```bash
for f in assets/images/*.png assets/images/*.jpg; do
  [ -f "$f" ] || continue
  w=$(sips -g pixelWidth "$f" | awk '/pixelWidth/{print $2}')
  h=$(sips -g pixelHeight "$f" | awk '/pixelHeight/{print $2}')
  if [ "$w" -gt 2000 ] || [ "$h" -gt 2000 ]; then
    sips -Z 2000 "$f"
  fi
done
```

读取图片后，识别以下信息：
- **页面类型**：列表页 / 表单页 / 详情页 / 弹窗 / 流程图 / 其他
- **主要功能**：该页面承载的核心业务功能
- **关键字段**：页面中可见的输入字段、展示字段、筛选条件
- **操作按钮**：页面中可见的所有操作入口（按钮、链接、图标）
- **状态/标签**：页面中展示的状态标识、标签分类
- **数据表结构**：若为列表页，识别表头列名和排序方式
- **识别限制**：图片中无法确认的内容（模糊、截断、遮挡等）

---

## 步骤 3：生成页面要点

在每张图片引用的下方，插入结构化要点块。格式如下：

```markdown
> 图N 页面要点
> - 页面类型：列表页
> - 主要功能：商品信息的查看、搜索、筛选和批量管理
> - 关键字段：商品名称、商品分类、SKU编号、价格、库存数量、状态
> - 操作按钮：新增、编辑、删除、导出、批量上架、批量下架
> - 状态/标签：上架（绿色）、下架（灰色）、待审核（橙色）
> - 数据表结构：表头含「商品名称|分类|SKU|价格|库存|状态|操作」，默认按创建时间倒序
> - 识别限制：右侧操作列按钮文字因截图分辨率模糊，无法确认是否包含「复制」按钮
```

**要求**：
- 每张图片必须生成对应要点块，不可跳过
- 要点内容基于图片实际可见信息，不可臆测
- 无法识别的内容必须在「识别限制」中明确标注
- Axure 画布截图（一张图包含多个 UI 状态）应分别描述每个状态区域

---

## 步骤 4：标准化图片引用

将所有 Obsidian 格式转为标准 Markdown 格式：

| 原始格式 | 转换后格式 |
|---|---|
| `![[商品列表页.png]]` | `![商品列表页](../../../assets/images/商品列表页.png)` |
| `![[Pasted image 20260325151102.png]]` | `![语义化中文描述](../../../assets/images/语义化中文描述.png)` |

**路径计算规则**：根据 PRD 文件所在深度计算到 `assets/images/` 的相对路径：
- `cases/prds/YYYYMM/*.md` → `../../../assets/images/`

**文件重命名**：无意义文件名（时间戳、UUID、MD5 哈希）必须重命名为中文语义化名称，反映图片实际内容。

---

## 步骤 5：需求澄清问答

扫描 PRD 全文，收集以下类型的不确定项：

| 类型 | 示例 |
|---|---|
| 字段定义缺失 | 「状态」字段有哪些可选值？ |
| 权限说明缺失 | 哪些角色可以执行「删除」操作？ |
| 交互逻辑不明 | 点击「保存」后是返回列表页还是停留在当前页？ |
| 边界条件未定义 | 「商品名称」最大输入长度是多少？ |
| 联动规则未说明 | 选择「分类A」后，「子分类」下拉框是否联动刷新？ |
| 异常处理未定义 | 网络超时时页面如何提示？ |

将所有问题统一收集后，以带选项的格式向用户提问：

```markdown
## 需求澄清（共 N 个问题）

### 问题 1：「审批状态」字段的可选值
- A. 待审批 / 审批中 / 已通过 / 已驳回（参考同类功能推测）
- B. 待提交 / 审核中 / 已通过 / 已退回 / 已撤销（更完整的状态流）
- C. 自行输入

### 问题 2：列表页默认排序规则
- A. 按创建时间倒序（最新优先）
- B. 按更新时间倒序
- C. 自行输入
```

**要求**：
- 每个问题必须提供至少 2 个候选答案 + 「自行输入」选项
- 候选答案应基于 PRD 上下文、同类产品经验或源码分析推导
- 关键问题（影响用例编写的）标注为「必答」

---

## 步骤 6：健康度预检

按以下清单逐项检查 PRD 完整性：

| 检查项 | 编码 | 说明 |
|---|---|---|
| 字段定义表 | W001 | 是否包含完整的字段名、类型、必填、校验规则 |
| 权限说明 | W002 | 是否说明各角色的操作权限 |
| 异常处理 | W003 | 是否定义了异常场景的系统行为 |
| 状态流转 | W004 | 涉及状态变更时是否给出状态机 |
| 接口定义 | W005 | 是否提供关键接口的入参出参 |
| 分页说明 | W006 | 列表页是否说明分页规则和默认条数 |
| 导航路径 | W007 | 是否明确各页面的菜单导航路径 |
| 数据格式 | W008 | 日期、金额、数量等字段的显示格式是否定义 |

输出格式：

```markdown
## 健康度预检报告

- 健康度评分：{{score}}/8
- 警告列表：
  - W001: 缺少字段定义表 — 列表页和表单页的字段类型、校验规则未定义
  - W003: 异常处理缺失 — 未说明网络超时、并发冲突等异常场景
  - W007: 导航路径未标注 — 无法确定「商品管理」在菜单树中的完整路径
```

---

## 输出要求

增强完成后输出：
1. **增强后的 PRD 文件**（原文件覆盖写入，文件名追加 `-enhanced` 后缀）
2. **页面要点数量统计**
3. **健康度预检报告**
4. **需求澄清问题列表**（若有）

将图片数量、要点数量、健康度警告列表写入状态文件，供后续节点使用。
