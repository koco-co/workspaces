# PRD 文档增强 Skill

本 Skill 解决 PRD 文档中图片无法被 AI 自动读取的问题，通过多模态能力逐张解析图片，生成结构化文本描述，使后续用例编写 Skill 能获取完整的 UI 信息。

**执行前必须阅读本文件和 `references/prd-template.md`。**

---

## 一、执行流程

```
Step 1: 定位 PRD 文件
Step 2: 扫描所有图片引用
Step 3: 定位图片文件
Step 4: 逐张读取图片（多模态）
Step 5: 生成结构化描述
Step 6: 图片重命名（语义化）
Step 7: 输出增强版 PRD
Step 8: 向用户展示增强摘要
```

---

## 二、Step 1: 定位 PRD 文件

接受以下输入：
- 文件绝对路径（如 `/Users/poco/Documents/DTStack/WorkSpaces/zentao-cases/...`）
- 相对路径（相对于 WorkSpaces 根目录）
- Story 目录 + PRD 编号（如 `Story-20260322/PRD-26`）

如用户未明确指定，询问 PRD 文件路径。

---

## 三、Step 2: 扫描图片引用

读取 PRD 文件，使用正则提取所有 Obsidian 图片引用：

```
![[Pasted image YYYYMMDDHHMMSS.png]]
![[任意图片名.png]]
```

统计图片总数，列出所有图片引用，向用户报告：
```
发现 N 张图片引用，开始逐张处理...
```

---

## 四、Step 3-4: 定位并读取图片

图片文件搜索顺序：
1. PRD 文件同级的 `resources/` 目录
2. PRD 文件同级的 `media/` 目录
3. PRD 文件同级目录（直接同级）
4. Obsidian vault 的全局 attachments 目录（如 `_resources/`）

使用 Read 工具读取图片文件（多模态能力自动解析图片内容）。

如果图片文件不存在，记录为「找不到文件」并继续处理下一张。

---

## 五、Step 5: 生成结构化描述

每张图片读取完毕后，立即生成结构化描述，格式如下：

```markdown
> **[AI图片描述]**
> 页面类型：列表页 / 表单页 / 详情页 / 弹窗 / 流程图
> UI 布局：[顶部搜索区 | 操作按钮区 | 数据列表区 | 分页区等]
> 按钮：【按钮名1】【按钮名2】...（精确到每个字）
> 表格列：列名1 | 列名2 | 列名3 | ...
> 搜索字段：字段名1 | 字段名2 | ...
> 表单字段：字段名1（文本）| 字段名2（下拉）| 字段名3（日期）
> 流程节点：节点1 → 节点2 → 节点3（仅流程图）
> 弹窗标题：xxx（仅弹窗）
> 状态/标签：状态值1 | 状态值2（如有）
> 其他关键信息：xxx
```

描述规则（严格执行）：
- 按钮名称必须精确，不得猜测
- 表格列名必须完整列出，不得省略
- 如图片模糊或无法识别，注明「图片质量不佳，部分内容无法识别」
- 禁止推断图片中不存在的内容

---

## 六、Step 6: 图片重命名（语义化）

将时间戳格式的图片名替换为语义化名称：

| 原名 | 语义化名称规则 |
|------|--------------|
| `Pasted image 20250101120000.png` | `<功能名>-<页面类型>.png` |

示例：
- `Pasted image 20260322143521.png` → `质量问题台账-列表页面.png`
- `Pasted image 20260322143600.png` → `质量问题台账-新增弹窗.png`

操作步骤：
1. 在 PRD 文件中替换引用名（`![[旧名]]` → `![[新名]]`）
2. 使用 Bash 工具重命名实际文件（`mv "旧名" "新名"`）
3. 如同名文件已存在，在新名后加 `-2`、`-3` 等后缀

---

## 七、Step 7: 输出增强版 PRD

输出文件命名规则：在原文件名后加 `-enhanced`，扩展名不变。

示例：
- 输入：`PRD-26-数据质量-质量问题台账.md`
- 输出：`PRD-26-数据质量-质量问题台账-enhanced.md`

输出位置：与原文件同级目录。

**增强内容（不删除原内容，只新增）：**
1. 在每张图片引用的下方插入描述块
2. 图片引用替换为语义化名称
3. 如原文缺失关键章节（字段定义表、交互规则），参考 `references/prd-template.md` 补充章节标题和空框架

**不做的事：**
- 不删除原文内容
- 不改变原文的段落顺序
- 不修改原文的文字表述

---

## 八、Step 8: 向用户展示增强摘要

```
PRD 增强完成！

📄 输出文件：<文件路径>

图片处理摘要：
- 共发现：N 张图片
- 成功读取：M 张
- 找不到文件：P 张（列出文件名）

关键信息提取：
- 发现 X 个功能页面
- 发现按钮：【按钮1】【按钮2】... 共 N 个
- 发现表单字段：N 个（含 M 个必填字段）

下一步建议：
确认增强结果无误后，可调用 test-case-generator 生成测试用例。
```
