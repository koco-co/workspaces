# 标准化 PRD 模板规范

PRD 增强后的输出文件（`*-enhanced.md`）应尽量符合以下结构。
注意：增强是在原文基础上补充，不应强行重构原文结构——结构调整只在明显缺失关键章节时才做。

---

```markdown
# [PRD编号] [功能名称]

## 1. 程序描述

[功能概述，说明本功能的业务背景和主要目标]

## 2. 权限控制

| 一级模块 | 二级模块 | 页面 | 权限项 | 类型 |
|---------|---------|------|-------|------|
| xxx | xxx | xxx | 查看/新增/编辑/删除 | 按钮权限/菜单权限 |

## 3. 流程逻辑

[流程说明 + 流程图（如有）]

## 4. 功能详细设计

### 4.1 [子功能1名称]

#### 页面说明

![[语义化图片名称.png]]

> **[AI图片描述]**
> 页面类型：列表页 / 表单页 / 详情页 / 弹窗
> UI 布局：[顶部搜索区域 / 操作按钮区域 / 数据列表区域 / 分页区域等]
> 按钮：【按钮名1】【按钮名2】...
> 表格列：列名1 | 列名2 | ...
> 搜索字段：字段名1 | 字段名2 | ...

#### 字段定义

| 字段名称 | 字段类型 | 是否必填 | 校验规则 | 枚举值/说明 |
|---------|---------|---------|---------|------------|
| xxx | 文本/下拉/日期/数字 | 是/否 | 最大X字符 / 格式要求 | 选项1/选项2 |

#### 交互规则

- [联动规则：选择A字段后，B字段自动回显...]
- [校验规则：提交时必填字段为空则...]
- [状态变更：操作成功后，状态从X变为Y...]

### 4.2 [子功能2名称]

...

## 5. 附录

[字段详细定义、数据逻辑、枚举值完整列表等]
```

## 图片描述格式规范

每张图片描述块的格式如下（插入在 `![[图片名]]` 下方）：

```markdown
> **[AI图片描述]**
> 页面类型：xxx
> UI 布局：xxx
> 按钮：【按钮名1】【按钮名2】
> 表格列：列1 | 列2 | 列3
> 搜索字段：字段名1 | 字段名2
> 表单字段：字段名1（类型：文本/下拉/日期）| 字段名2
> 流程节点：节点1 → 节点2 → 节点3（如为流程图）
> 其他关键信息：xxx
```

描述原则：
1. 必须提取所有可见的按钮文字（精确到每个字）
2. 必须列出所有表格列名
3. 必须列出所有搜索框字段名
4. 必须列出所有表单字段名及控件类型
5. 如是流程图，列出所有节点名称和流转关系
6. 如是弹窗，说明弹窗标题和包含字段
