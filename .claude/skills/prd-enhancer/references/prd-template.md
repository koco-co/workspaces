# 标准化 PRD 模板规范

PRD 增强后的输出文件（`*-enhanced.md`）应尽量符合以下结构。
注意：增强是在原文基础上补充，不应强行重构原文结构——结构调整只在明显缺失关键章节时才做。

---

## Front-Matter 模板

```yaml
---

dev_version: "6.3岚图定制化分支"

  - 关键词1

  - "W001: 缺少字段定义表"

  - ".repos/DTStack/dt-center-assets"

---
```

## 图片描述格式规范

每张图片描述块的格式如下（插入在 `![描述](路径)` 图片引用下方）：

```markdown
> **[AI图片描述]**
> 页面类型：xxx
> UI 布局：xxx
> 按钮：【按钮名1】【按钮名2】
> 表格列：列1 | 列2 | 列3
> 搜索字段：字段名1 | 字段名2
> 表单字段：字段名1（类型：文本/下拉/日期）| 字段名2
> 流程节点：节点1 → 节点2 → 节点3（如为流程图）
> 其他关键信息：xxx
```

描述原则：
1. 必须提取所有可见的按钮文字（精确到每个字）
2. 必须列出所有表格列名
3. 必须列出所有搜索框字段名
4. 必须列出所有表单字段名及控件类型
5. 如是流程图，列出所有节点名称和流转关系
6. 如是弹窗，说明弹窗标题和包含字段
