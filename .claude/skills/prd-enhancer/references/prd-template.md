# 标准化 PRD 模板规范

PRD 增强后的输出文件应尽量符合以下结构，增强文件以 front-matter 状态识别，不依赖文件名后缀。
注意：增强是在原文基础上补充，不应强行重构原文结构——结构调整只在明显缺失关键章节时才做。

---

## Front-Matter 模板

```yaml
---
prd_name: "需求名称（与蓝湖一致）"
description: "一句话描述（≤60字）"
prd_id: 10287
prd_version: v2.0.0
prd_source: "cases/prds/YYYYMM/【功能名】需求标题.md"
prd_url: "https://lanhuapp.com/..."
product: orders
dev_version: "release/v2.0"
tags:
  - 关键词1
create_at: "YYYY-MM-DD"
update_at: "YYYY-MM-DD"
status: 已增强
health_warnings:
  - "W001: 缺少字段定义表"
repos:
  - ".repos/orders-service"
case_path: ""
---
```

状态字段读写规则见 [frontmatter-status-map.md](./frontmatter-status-map.md)。

## 页面要点格式规范

每张图片引用下方统一插入 `图N 页面要点`，具体块结构与字段定义见 [page-insight-format.md](./page-insight-format.md)。

描述原则：
1. 必须提取所有可见的按钮文字（精确到每个字）
2. 必须列出所有表格列名或列表关键字段
3. 必须列出所有搜索 / 筛选条件
4. 必须列出所有输入项及可识别控件类型
5. 如是流程图，列出所有节点名称和流转关系
6. 如存在识别盲区或图片模糊，统一写入「识别限制」

## 缺失章节处理规则

执行增强时，如原始 PRD 中以下章节内容不明确或不存在，**必须暂停并向用户提问**，等待回复后再填写；禁止使用 `TODO` 占位符替代确认；若用户明确表示不涉及，则填入 `> 本功能不涉及此章节内容`：

| 章节 | 缺失时必须确认的情况 |
| ---- | -------------------- |
| 四、字段联动规则 | 存在多个表单字段，但未描述任何联动行为时 |
| 五·5.1 提交校验规则 | 存在表单提交功能，但未列出任何校验规则时 |
| 六、流程与状态机 | 需求描述中提及审批、流程、状态流转，但无状态图说明时 |
| 七、异常与边界场景补充 | 功能较复杂（超过 3 个表单字段或 2 个状态），但未列出任何异常场景时 |
| 八、本期不涉及的功能 | 存在相关联功能模块，但未明确说明本次是否涵盖时 |

对于能从原始 PRD 推断的内容（如字段约束、枚举值），优先使用推断，并标注 `[PRD 未说明，基于推断]`，**无需**向用户确认。

## 填写指南

为什么要这么详细？此模版的每个章节都直接对应测试用例生成的关键输入：

| 模版章节          | 解决的用例生成问题                                     |
| ----------------- | ------------------------------------------------------ |
| 三、页面元素清单  | 确保用例中的按钮名称、字段名称与实际一致，不凭空杜撰   |
| 三·3.2 表单字段   | 提供字段类型和约束，让用例包含具体测试数据而非模糊描述 |
| 三·3.3 枚举选项值 | 用例中的下拉选择能写出真实选项值                       |
| 四、字段联动规则  | 生成联动验证用例，避免遗漏自动回显场景                 |
| 五、校验逻辑      | 每条校验规则直接映射为一条异常用例                     |
| 六、流程与状态机  | 生成流程提交、各节点审批通过/驳回的完整用例链          |
| 七、异常场景补充  | 覆盖你预判的高风险边界场景                             |
| 八、明确排除      | 防止 AI 超出本次需求范围编写多余用例                   |

**填写原则：**

1. 名称一致性 — 按钮叫【提交】就写【提交】，不要写【确认提交】或【保存】
2. 穷举选项值 — 下拉有 5 个选项就全列出来，不要写「等」
3. 约束要量化 — 写「1-100字符」而不是「不能太长」
4. 联动要成对 — 写清楚「选了什么 → 哪个字段 → 变成什么」
5. 不确定就询问 — 遇到无法从 PRD 或图片推断的关键信息时，**必须暂停并向用户提问**，不要写 `TODO` 占位符；只有用户明确表示"无需确认 / 不涉及"时才允许填入 `> 本功能不涉及此章节内容`
