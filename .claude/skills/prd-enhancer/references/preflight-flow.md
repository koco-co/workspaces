# 前置检查链路

本文件定义 `prd-enhancer` 在进入主流程前必须完成的 5 个检查节点。

## 链路总表

1. 来源识别
2. 正式化门禁
3. 定位目标 PRD
4. 增量检测
5. 独立调用需求澄清

## 节点说明

### 1. 来源识别

- 判断输入是本地正式 PRD、上游 formalized 文档，还是外部平台原始文本。
- 若无法确认输入类型，不进入主流程。

### 2. 正式化门禁

- 当 `config.repos` 非空且输入为外部平台原始文本时，必须先进入 `prd-formalizer`。
- 当 `config.repos` 为空，或输入本身已是正式 PRD 时，直接继续下一节点。

### 3. 定位目标 PRD

- 支持绝对路径、相对路径、模块/版本/编号三类输入。
- 必须定位到唯一文件；若存在多候选，应由上游编排或调用参数完成消歧。

### 4. 增量检测

- 扫描目标文件同目录的 Markdown 文件 front-matter。
- 读取时兼容 `enhanced` / `已增强`；若已有增强文件且原文未变化，则直接复用；若原文变化，则仅对变更章节执行增量处理。

### 5. 独立调用需求澄清

- 仅在用户直接触发 `prd-enhancer` 时，根据前置检查结果决定是否补做需求澄清。
- 独立调用需求澄清的评估维度与权重，统一参考权威来源 `../../test-case-generator/references/elicitation-dimensions.md`，本 Skill 不重复定义。
- 若由 `test-case-generator` 自动调用，则跳过该节点。
- 本节点只决定是否补齐上游信息，不在本 Skill 文档中定义额外结果章节格式。
