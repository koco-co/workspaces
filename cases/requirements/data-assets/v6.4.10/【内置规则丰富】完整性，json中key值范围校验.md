# 1. 需求背景

本需求属于「数据资产 6.3岚图定制化分支」迭代，蓝湖文档：数据资产V6.4.10（#15693）。

**变更性质：** 🆕 新增功能 — 完整性校验新增「key范围校验」统计函数类型。

完整性校验目前支持对字段做值存在性、空值等基础校验。本需求进一步扩展完整性校验能力，使其支持对 **json 或 string 类型字段**中的 key 名称进行范围校验（即检查 json 字段内的 key 是否属于预设范围）。

# 4. 功能详细设计

## 4.1 规则配置 - 统计函数选择

### 4.1.1 入口页面

**页面路径：** 数据质量 → 规则配置 → 新建规则集 / 编辑规则集 → 规则编辑步骤

**后端接口：** `POST /api/v1/valid/function/getFunctions`（获取统计函数列表，按规则类型过滤）

### 4.1.2 字段规则

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 规则类型 | 下拉单选 | 选择「完整性校验」（`COMPLETE = 1`）后，统计函数列表过滤出完整性相关函数 |
| 统计函数 | 下拉单选 | 新增选项：**key范围校验** |
| 字段 | 下拉选择 | **选择「key范围校验」后：仅支持单选字段**（`isMultipleSupport = 0`）；仅展示数据类型为 json 或 string 的字段 |
| 校验方法 | 下拉单选 | 可选值：**包含** / **不包含** |
| 校验内容 | 多选选择器 | 从「json格式配置」模块取 key 名，支持多选 / 全选，支持层级选择，支持输入搜索 |
| 过滤条件 | 文本输入 | 可选，WHERE 条件 SQL |
| 强弱规则 | 单选 | 强规则 / 弱规则 |
| 规则描述 | 文本输入 | 可选 |

### 4.1.3 交互规则

1. **悬浮提示（统计函数选项旁）：**
   - 悬浮「key范围校验」时展示提示文案：`"当选择key范围校验时，字段仅支持单选"`

2. **字段选择限制：**
   - 选择统计函数为「key范围校验」后，「字段」下拉框仅展示数据类型为 `json` 或 `string` 的字段
   - 字段选择框改为单选模式（不支持多选）

3. **校验方法切换：**
   - 默认值：包含
   - 切换至「不包含」时，校验逻辑相反（字段 key 不应出现在配置的范围内）

## 4.2 规则配置 - 校验内容选择器

### 4.2.1 数据来源

校验内容的 key 列表来源于「通用配置 - json格式配置」模块维护的数据。选择列表取 key 名（支持层级 key，如 `key1-key2` 表示 key1 下的子 key2）。

### 4.2.2 选择器行为

| 行为 | 规则 |
|------|------|
| 数据加载 | 默认加载前 **200** 条 key 展示（key 数量多时） |
| 多选 | 支持多选，支持勾选「全部」一键选中所有当前层级 key |
| 层级选择 | 勾选父 key 时仅对当前层级生效（不自动递归勾选子 key） |
| 搜索 | 选择框内支持输入关键词过滤 key 列表 |
| 回显格式 | 已选 key 路径用 `-` 连接层级，多个 key 用 `;` 分隔。示例：`key1-key2;key11-key22` |

### 4.2.3 key 层级示例

```
全部
├── key1（中文名称）
│   ├── key11（中文名称）
│   └── key22（中文名称）
└── key2（中文名称）
    ├── key11（中文名称）
    ├── key22（中文名称）
    └── key33（中文名称）
key3（中文名称）
```

> **注意：** 配置后按照层级进行校验，key名需要按照层级匹配是否存在key信息；需考虑 key 数量几千个的情况（前200条默认展示，其余需搜索）。

# 6. 前置条件说明

测试本需求前，需要：

1. 在「通用配置 - json格式配置」模块（#15696）中维护 key 数据
2. 准备包含 json 类型字段的数据表（以下为示例 SQL）：

```sql
-- Doris 3.x 数据准备示例
DROP TABLE IF EXISTS test_db.test_json_table;
CREATE TABLE test_db.test_json_table (
    id INT,
    name VARCHAR(100),
    json_field STRING
) ENGINE=OLAP
DUPLICATE KEY(id)
DISTRIBUTED BY HASH(id) BUCKETS 1;

INSERT INTO test_db.test_json_table VALUES
(1, 'record1', '{"key1": "val1", "key2": "val2"}'),
(2, 'record2', '{"key1": "val1", "key3": "val3"}'),  -- key2 缺失，用于测试不通过场景
(3, 'record3', '{"key1": "val1", "key2": "val2", "key4": "val4"}');  -- 含额外 key
```
