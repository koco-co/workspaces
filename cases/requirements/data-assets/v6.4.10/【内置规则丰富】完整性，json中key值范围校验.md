# 【内置规则丰富】完整性，json中key值范围校验

## 开发版本

6.3岚图定制化分支

## 需求内容

完整性校验支持对json类型的字段做key值范围的校验。

---

## 规则配置

### 统计函数新增：key范围校验

- 支持选择「key范围校验」统计函数
- 悬浮提示文案："当选择key范围校验时，字段仅支持单选"
- 选择key范围校验后，字段选择仅支持单选

### 校验方法

- 支持选择：**包含** / **不包含**

### 校验内容

- 选择列表来源：「json格式配置」模块维护的数据列表，取key名
- 支持多选 / 选择全部
- 选择框内支持输入查询（搜索过滤）
- 若key数据量多，默认加载前200条展示
- 勾选仅对当前层级生效
- 回显格式：`key1-key2;key11-key22`（父key-子key，分号分隔）

### 示例 key 列表（来自json格式配置维护）

```
全部
key1（中文名称）
key2（中文名称）
key3（中文名称）
key11（中文名称）
key22（中文名称）
key33（中文名称）
```

---

## 注意事项

1. 配置后需要按照层级进行校验，key名需要按照层级匹配是否存在key信息；【要考虑key数量几千个的情况】
2. 支持数据源：Doris3.x、SparkThrift2.x、Hive2.x
3. key范围校验仅支持数据类型为 **json**、**string** 的字段

### 质量报告展示

| 规则类型 | 规则名称 | 字段名称 | 字段类型 | 质检结果 | 未通过原因 | 详情说明 | 操作 |
|--------|---------|---------|--------|--------|---------|---------|------|
| 完整性校验 | key范围校验 | xxxx | json | 校验通过 | -- | 符合规则key范围包含/不包含"key1-key2;key11-key22……" | -- |
| 完整性校验 | key范围校验 | xxxx | json | 校验不通过 | key范围校验未通过 | 不符合规则key范围包含/不包含"key1-key2;key11-key22……" | 查看详情 |

### 明细查看（校验不通过时）

- 针对「校验不通过」的规则，支持查看明细
- 明细标题文案修改为：**查看"完整性校验-key范围校验"明细**
- 记录不符合要求的数据
- 数据列表保留全部字段，**校验字段标红展示**
- 下载明细数据中**校验字段也标红展示**

### 其他规则行为

- 针对「校验通过」的规则，不记录明细数据
- 针对「校验失败」的规则，支持查看日志
