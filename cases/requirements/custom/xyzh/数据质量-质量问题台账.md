# PRD-26 数据质量-质量问题台账

## 1. 程序描述

本模块用于管理质量问题台账。支持质量问题的创建、编辑，实现质量问题从发现到解决的管理。台账数据来源于数据运营平台，可与外部处理系统联动。

![质量问题台账模块整体页面概览](../../../../../assets/images/质量问题台账模块整体页面概览.png)

> **[结构化观察]**
> - 页面定位：截图显示当前顶部页签为「质量问题台账」，面包屑为「数据质量 / 质量问题台账 /」。
> - 主体布局：搜索筛选区 → 列表标题「质量问题列表」→ 列表操作按钮「新增问题」→ 数据列表区。
> - 列表可见列：问题名称 / 问题编号 / 问题发起人 / 问题发起人部门 / 问题描述 / 是否进行处理 / 问题状态 / 操作。
> - 操作列可见按钮：详情 / 编辑 / 删除。
> - 截图中可见状态示例：未处理 / 处理中 / 已解决。
> - 截图中的不同状态行按钮组合并不完全一致，至少可见「详情」「编辑」「删除」三类按钮；其中已解决示例行未识别到「编辑」按钮。
> - 原文权限表写作「数据标准 / 落标问题台账 / 创建」，截图实际呈现为「数据质量 / 质量问题台账 / 新增问题」，此处保留差异说明，不擅自统一。

#### 4.1.3 新增问题录入面板

该页面用于创建新的质量问题。用户可填写问题名称、描述、行动编号、选择是否进行处理，填写处理人及所属部门等等。

![质量问题新增表单页](../../../../../assets/images/质量问题新增表单页.png)

> **[结构化观察]**
> - 呈现方式：截图中的录入区域覆盖在列表内容之上，左侧仍能看到列表标题与部分列表数据，更接近覆盖式录入面板/侧滑编辑区域，而非完全独立的空白页面。
> - 面板顶部可见分组标题「质量问题基本信息」，右上角可见「保存」按钮。
> - 基础信息区可识别字段：问题发起时间 / 问题发起人 / 问题名称 / 问题编号 / 是否进行处理 / 行动编号 / 计划完成时间 / 行动描述 / 行动状态 / 责任人 / 责任部门 / 行动报告生成时间 / 行动报告结束时间 / 问题描述。
> - 「是否进行处理」处可见单选项「是 / 否（需在问题描述阐述原因）」。
> - 「行动编号」右侧可见「查询」按钮；计划完成时间、行动描述、行动状态、责任人、责任部门等字段以回显值的形式同屏展示。
> - 面板下方存在「关联的质量检查任务清单」区域，区头可见「新增」按钮；表格列可识别为操作 / 序号 / 质量问题编号 / 数据质量检查任务名称 / 报警规则 / 检查维度。
> - 任务清单下方可见「批量删除」「确认删除」按钮。
> - 底部存在「附件」区域与「上传文件」按钮；截图提示仅支持 `<200mb` 的 `.pdf/.doc/.docx/.ppt/.pptx/.csv/.xls/.xlsx` 文件上传。
> - 当前截图不是纯空白新建态，而是带示例数据/回显数据的录入态，因此不扩写“初始字段全部为空”“保存成功提示”等未被截图证明的细节。

##### 字段定义（主表字段）

| 字段名称 | 字段类型 | 是否必填 | 校验规则 | 说明 |
| --- | --- | --- | --- | --- |

##### 交互规则

- 行动编号支持点击「查询」后回显行动描述、计划完成时间、责任部门、责任人、行动报告生成时间、行动报告结束时间、行动状态等信息。
- 「是否进行处理」为「否」时，问题描述需要填写不处理原因；截图中已直接展示该约束提示。
- 关联质量检查任务清单与附件区域在录入界面中同屏存在，不需要切换到其他页面才能维护。

当前截图仅展示一条「处理中」示例记录，未明确标出哪些字段被锁定，因此本增强版只保留原文级别描述，不扩写字段级编辑矩阵。
