【数据目录管理-管理域目录管理】

能力拓展：导入新增L3节点时，自动在主数据标准标准-L4子表中创建该L3下属的L4默认实体。

**定位方式**：搜索 '20260305' 定位变更内容

## 一、功能变更说明

### 1.1 变更模块

**模块路径**：数据目录管理 → 管理域目录管理 → 导入新增

### 1.2 变更内容

#### 导入新增功能 - 自动创建L4默认实体

**【20260305】更新**

**L4表需求（见附录3）**

## 二、功能详细设计

### 2.1 导入新增流程

用户可以且仅可以通过导入功能新增目录节点。

本功能只用于新建全新的节点，不可以用于现有节点属性的批量维护。

#### 导入流程图

![导入新增流程](../../../../../asset/assets/images/导入新增流程.png)

### 2.2 导入处理步骤

程序按照下图依次进行对应方法：

#### ①模板正确性、行数上限校验

本步骤旨在确定模板表头的可识别性，以及导入数据规模不会过大。

1. 表头完整性（不通过时报错'模板失败失败'）
2. 记录行数（>5000条时报错'超出导入记录上限，最多导入5000条数据'）

#### ②解析并分层

本步骤旨在识别每一行记录的实际节点层级并分组，以便后续可按照层级进行从根到末的校验与创建。

1. 'L3业务对象名称'被填写时，适用L3校验规则。
2. 'L2主题域名称'被填写，且'业务数据负责人工号'之右的列全部为空时，适用L2校验规则
3. 'L1主题域分组名称'被填写，且'业务数据Owner工号'之右的列全部为空时，适用L1校验规则
4. 不符合以上3个条件的都记为导入失败记录。

记录失败条数。

#### ③重复数据过滤

本步骤旨在通过编号进行去重。

1. 对于适用L3校验规则的行，L1、L2、L3本级编码列完全一致的记录。只留第一条，其他数据过滤，记为失败。
2. 对于适用L2校验规则的行，L1、L2本级编码列完全一致的记录。只留第一条，其他数据过滤，记为失败。
3. 对于适用L1校验规则的行，L1本级编码列完全一致的记录。只留第一条，其他数据过滤，记为失败。

记录失败条数。

#### ④L1记录合法性校验

本步骤旨在保障L1数据的合法性。

1. 完整性：'L1主题域分组本级编码'、'L1主题域分组名称'、'业务数据owner工号'全部不为空。
2. 用户有效性：'业务数据owner工号'、'业务主管部门数据负责人工号'存在。
3. 合规性：'L1主题域分组本级编码'、'L1主题域分组名称'位数与要求一致、且名称、编码在同父级中不重复。

过滤没有通过校验的记录。记录失败条数。

#### ⑤新建L1数据

本步骤旨在创建前一步合法数据。这也方便了后续下属节点的校验与创建。

新建：上级节点ID(固定为'管理域''M')、L1主题域分组名称、L1主题域分组本级编码、业务数据owner工号

#### ⑥L2记录合法性校验

本步骤旨在保障L2数据的合法性。

1. 完整性：'L1主题域分组本级编码'、'L2主题域名称'、'L2主题本级编码'、'业务数据负责人工号'全部不为空。
2. 父级、用户有效性：'L1主题域分组本级编码'、'业务数据负责人工号'存在。
3. 合规性：'L2主题域名称'、'L2主题域本级编码'位数与要求一致、且名称、编码分别在同父级中都不重复。

过滤没有通过校验的记录。记录失败条数。

#### ⑦新建L2数据

本步骤旨在创建前一步合法数据。这也方便了后续下属节点的校验与创建。

新建：上级节点ID、L2主题域名称、L2主题本级编码、领域数据Owner工号、领域数据负责人工号。

#### ⑧L3记录合法性校验

本步骤旨在保障L3数据的合法性。

1. 完整性：'L1主题域分组本级编码'、'L2主题本级编码'、'L3业务对象名称'、'L3业务对象英文名称'、'业务对象定义'、'业务对象数据标准责任人工号'全部不为空。
2. 父级、用户有效性：'L1主题域分组本级编码'、'L2主题本级编码'、'数据责任人工号'存在。
3. 合规性：'L3业务对象名称'、'业务对象定义'、'L3业务对象英文名称'、'L3业务对象英文简称'内容位数与要求一致、且名称在同父级内不重复。

过滤没有通过校验的记录。记录失败条数。

#### ⑨新建L3数据

新建：上级节点ID、L3业务对象名称、L3业务对象本级编码（生成）、L3业务对象英文名称、L3业务对象英文简称、业务对象定义、领域数据负责人工号、是否已认责(=false)

#### ⑩【新增】自动创建L4默认实体

**【20260305】更新**

在创建增量L3节点完成后，自动创建该L3的下属L4。

**L4默认实体信息**：

| 属性 | 值 |
|------|------|
| L4名称 | 默认实体 |
| L4本级编码 | E000 |
| 表名 | Default |

> L4表需求（见附录3）

#### ⑪返回导入结果

本步骤旨在所有创建完毕后，告知本次导入的影响。

toast提示：

'导入完成，其中成功X条，识别失败X条，重复X条，L1失败X条，L2失败X条，L3失败X条，其他失败X条'

## 三、数据结构

### 3.1 L4默认实体

| 字段 | 值 | 说明 |
|------|------|------|
| 上级节点ID | 新创建的L3节点ID | 关联父节点 |
| L4名称 | 默认实体 | 固定值 |
| L4本级编码 | E000 | 固定值 |
| 表名 | Default | 固定值 |

### 3.2 性能要求

| 指标 | 要求 |
|------|------|
| 查询响应时间 | ≤ 0.5s |
| 同步时间 | ≤ 5分钟 |
| 导入处理完毕时间 | < 30s |

---

## 四、影响范围

### 4.1 影响的功能点

- 管理域目录管理 - 导入新增功能
- 主数据标准 - L4子表

### 4.2 业务影响

- 每个新增的L3节点将自动拥有一个L4默认实体
- 无需用户手动创建L4默认实体，减少操作步骤
- 确保数据结构的完整性

### 4.3 注意事项

- L4默认实体仅在导入新增L3节点时自动创建
- 手动创建L3节点时是否同步创建L4默认实体需另行确认
- L4默认实体的属性值（名称、编码、表名）为固定值，不可配置

## 五、验收标准

| 序号 | 验收项 | 预期结果 |
|------|--------|----------|
| 1 | 导入新增L3节点成功 | 自动创建对应的L4默认实体 |
| 2 | L4默认实体名称 | 等于"默认实体" |
| 3 | L4默认实体本级编码 | 等于"E000" |
| 4 | L4默认实体表名 | 等于"Default" |
| 5 | L4默认实体与L3的关联关系 | 正确关联到父L3节点 |
| 6 | 导入失败时 | 不创建L4默认实体 |
