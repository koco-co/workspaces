# 【通用配置】json格式配置

## 需求概述

- 新增 `json格式校验管理` 页面，用于维护 json key 配置树。
- 配置内容包括：`key`、`中文名称`、`value格式`、`数据源类型`。
- 本需求是 15693、15694 的基础能力来源。

## 依赖关系

- 下游依赖：
  - 15693 `key范围校验` 的校验内容来源于本页面维护的 key 数据。
  - 15694 `json value 格式校验` 的校验 key 和 value 格式来源于本页面维护的数据。

## 页面要点

- 支持按 `key` 模糊搜索，且支持搜索到**子层级 key**。
- 列表字段包含：`key`、`中文名称`、`value格式`、`数据源类型`、`创建人/更新人`、`创建时间/更新时间`、`操作`。
- 对存在子层级的记录展示 `+` 按钮，下钻查看下一层级；不存在子层级时不展示。
- 展示条数按**最外层级**统计。
- 最多支持 **5 个层级**；第 **5 层**不支持 `新增子层级`。
- 单行操作：`编辑`、`新增子层级`、`删除`
- 批量操作：`批量删除`、`导入`、`导出`、`新增key`

## 字段定义

| 字段 | 是否必填 | 规则 |
| --- | --- | --- |
| key | 是 | 最大 255 字符；无需重复性校验 |
| 中文名称 | 否 | 最大 255 字符 |
| value格式 | 否 | 最大 255 字符；支持输入正则表达式 |
| 数据源类型 | 是 | 单选；默认 `sparkthrift2.x`；可选 `sparkthrift2.x` / `hive2.x` / `doris3.x` |

- 配置了 `value格式` 时，支持对正则内容进行测试；未配置时不展示测试数据字段。

## 删除与批量删除

- 单条删除提示：
  - `请确认是否删除key信息，若存在子层级key信息会联动删除`
- 批量删除提示：
  - `请确认是否批量删除key信息，若存在子层级key信息会联动删除`

## 导入规则

- 导入模板名：`json_format_import_template.xlsx`
- 错误文件名：`json_format_error_20240520.xlsx`
- 点击【确定】后先进行内容校验。
- 校验失败提示文案：
  - `导入表格中存在错误数据，请检查后重新导入，点击导出错误文件。`
- 模板结构：
  - 一个 xlsx 文件包含 **5 个 Sheet**，命名依次为：`一层`、`二层`、`三层`、`四层`、`五层`
  - `一层` Sheet 字段：`key`、`中文名称`、`value格式`
  - `二层`~`五层` Sheet 字段：`上一层级的key名`、`key`、`中文名称`、`value格式`
- 必填规则：
  - `一层`：`key` 必填；`中文名称`、`value格式` 非必填
  - `二层`~`五层`：`上一层级的key名`、`key` 必填；`中文名称`、`value格式` 非必填
- 校验规则：
  1. key 名称过长 / 必填项未填写 / 上一层级 key 无法找到时，对错误数据标红并批注原因。
  2. **全部数据均正确后才允许导入**。
  3. 导入数据均为**新增逻辑**。
  4. 若上一层级存在重复 key 名，默认取**第一个 key** 作为上一层级。

## 导出规则

- 支持列表全量导出与筛选导出。
- 文件命名规则：`json_format_+日期`，示例：`json_format_20260323`
- 导出确认提示：
  - `请确认是否导出列表数据`

## 重点测试关注点

1. 千级 key、多层级数据下，搜索、回显、层级展开是否正确。
2. 第 5 层限制、级联删除、批量删除是否符合预期。
3. key 配置已被质量任务引用后，若页面再次删除 key，历史规则是否仍正常回显，历史任务是否仍正常运行。
4. 5 个 Sheet 的导入模板、字段必填关系、重复上一层级 key 取第一个的规则是否落地。
