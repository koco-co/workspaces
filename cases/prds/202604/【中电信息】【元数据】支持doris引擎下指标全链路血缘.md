# 【中电信息】【元数据】支持doris引擎下指标全链路血缘

## 需求概述

- 蓝湖页面范围：17「15549【中电信息】【元数据】支持doris引擎下指标全链路血缘」与 18「指标全链路血缘」。
- 需求性质：**修复 + 能力补齐**。已有标品 5.3 的指标全链路血缘只支持 Trino 指标场景，本次需恢复并扩展到 Doris 指标场景。
- 核心目标：打通**表 → 指标**、**字段 → 指标字段**的血缘展示，并解决同物理表因离线侧与指标侧使用不同 meta 数据源而产生的链路割裂问题。

## 页面一：15549【中电信息】【元数据】支持doris引擎下指标全链路血缘

### 页面要点

- 历史能力支持链路：离线任务 → 表 → 指标，但旧实现将指标引擎写死为 Trino。
- 当前需求修复 Doris 指标场景：当离线侧和指标侧都接入 Doris，但通过不同 meta 数据源登记同一物理库表时，血缘图中需要识别并合并。
- 合并前现象：同一张表会以 `Doris2.x meta1`、`Doris2.x meta2` 两条记录存在，导致离线链路和指标链路断开。
- 合并后期望：
  - 从 `meta1` 源（或离线任务）进入表 B 血缘时，能够继续关联出指标血缘；
  - 从 `meta2` 源（或指标）进入表 B 血缘时，能够继续关联出离线任务血缘；
  - 展示上体现为同一链路节点兼容 `meta1 / meta2` 两个来源。

### 业务规则

1. “同一张表”的判断口径不是单纯按数据源记录，而是按 Doris 引擎下实际同库同表识别。
2. 由于已移除“相似数据源”逻辑，本次需要在血缘侧补足“多 meta 来源合并”的识别与展示。
3. 表级血缘与字段级血缘都要遵循相同的合并逻辑。
4. 字段血缘回归要求与表级一致：字段链路也要在 `meta1 / meta2` 间打通，不允许只修复表级。

### 页面示意补充

- 页面重点不是新增操作表单，而是链路修复逻辑与血缘图展示结果。
- 从用户补充截图可见，页面使用“现有血缘信息”与“合并后血缘模块展示”对比说明：
  - 表级示例：表 A → 离线任务 → 表 B → 指标；
  - 字段级示例：表 A.字段 a → 表 B.字段 b → 表 C.字段 c。

## 页面二：指标全链路血缘

### 页面要点

- 整体目标：实现表到指标的血缘链路打通，用户能够在数据资产侧查看数据表到指标之间的完整血缘关系。
- 指标生成方式覆盖两类：
  - **向导模式**：原子指标、派生指标、复合指标；
  - **脚本模式**：自定义 SQL 指标。
- 指标平台发生删除、下线等变动后，资产平台血缘视图需要同步更新。
- 需要支持字段血缘展示。
- 当指标血缘数量大于 1 时，需要展示特殊 icon。

### 业务规则

#### 1. 表到指标血缘解析

1. 需要根据指标平台“指标的生成”记录建立【表 → 指标】血缘关系。
2. 向导模式下，指标所选表即为血缘上游表；派生、复合指标基于原子指标继续展开。
3. 脚本模式下，需要基于查询主体 SQL 解析上游表与下游指标关系。
4. 脚本模式解析依赖业务中心 SQLParser 扩展能力，不再仅支持 `create`、`insert`、`truncate` 等语句。

#### 2. 自定义 SQL 指标约束

1. SQL 语句按 TrinoSQL 语法编写。
2. 一段 SQL 可以生成多个指标。
3. 支持动态参数表达式。
4. 支持聚合函数、子查询、逻辑运算符。
5. 不支持 DDL 操作。

#### 3. 血缘图展示与交互

1. 渲染逻辑与原先血缘图保持一致，默认展示一层，支持 `+` 展开、`-` 收起。
2. 以指标为主体展示血缘时，默认仅展示一层上游表；若上下游仍存在血缘，允许逐级展开到三级。
3. 右键菜单需要恢复并支持：
   - 添加上下游血缘
   - 查看当前节点血缘
4. 右上角小圆点只在“手动添加产生的血缘”节点上显示，用于删除节点；原生血缘不展示，指标节点也不展示。
5. 页面右上角开启“血缘类型筛选”，筛选项包含：
   - 表
   - 离线任务
   - 实时任务
   - API
   - 标签
   - 指标
6. 默认所有维度选中；当前进入的维度必须保持选中且不可取消。

#### 4. 多表生成同一指标

1. 单例模型：表后面新增一节指标血缘。
2. 星型/雪花模型：存在多表生成同一指标的情况。
3. 默认展示当前表进入的主干血缘，节点点击后可展开所有表生成该指标的血缘情况。
4. 血缘数量大于 1 的指标展示特殊 icon，提示存在多条来源链路。

#### 5. 字段级血缘

1. 当前节点高亮并居中显示，展示全部字段。
2. 仅有血缘关系的字段左侧展示血缘图标。
3. 点击字段后默认展开上下游一级字段血缘；若上下游表仍有血缘，支持继续展开到三级。
4. 展开的上下游节点仅展示有血缘关系的字段。
5. 支持逆向血缘：下游存在逆向字段血缘时，可继续展开/收起。
6. 原子指标不落表，需要在创建原子指标时记录字段信息，用于后续字段血缘展示。

### 页面示意补充

- 用户补充截图显示页面处于 DataAssets 系统的数据地图/血缘查看场景。
- 页面存在“表级血缘 / 字段级血缘”切换。
- 图中出现表节点、离线任务节点、指标节点、字段节点以及多来源展开示意。
- 可见右键菜单、筛选区域、指标特殊标识与字段展开效果。

## 需求澄清结果

- 本轮已结合蓝湖 17/18 页、用户补充截图、源码分支 `test_6.3.x_zdxx_0323` / `release_6.3.x_zdxx_0323` 做交叉核对，当前信息足以直接生成测试用例，不再继续追加用户问答。
- 已确认的实现入口与边界：
  - 资产侧主入口为【数据资产 → 数据地图 → 表详情 → 血缘关系】，核心组件为 `MetaDataDetails -> KinShip`。
  - 指标侧存在辅助校验入口【指标详情 → 血缘关系】，接口为 `/api/easyIndex/index/home/queryBloodRelationByIndexId`。
  - Doris2.x / Doris3.x 在元数据详情页均开放 `DATA_BLOODLINE` Tab。
  - 手动添加/删除仅作用于手工血缘节点；原生血缘节点与指标节点不展示删除圆点。
- 非阻断遗留确认项：
  - 历史数据是否需要一次性补数/回刷；
  - 指标删除、下线后的资产侧同步时效与触发方式。

## 回归与测试关注点

### P0 主路径

1. Doris 指标场景下，表级血缘能够从离线表链路正常打通到指标。
2. Doris 指标场景下，字段级血缘能够正常展示并展开上下游。
3. 同一物理表分别以 `meta1`、`meta2` 记录存在时，血缘图可按期望合并。
4. 从离线侧入口与指标侧入口进入，均能看到完整链路。

### P1 核心规则

1. 向导模式下原子、派生、复合指标的血缘解析。
2. 脚本模式下自定义 SQL 指标的血缘解析。
3. 单例模型与星型/雪花模型的差异化展示。
4. 类型筛选、右键菜单、手动添加/删除血缘、逆向血缘展开。

### P2 扩展与回归

1. 指标删除、下线后的血缘视图同步。
2. 历史数据兼容与结果表展示。
3. 血缘数量 > 1 时的特殊 icon 展示。

## 待澄清事项

1. 脚本模式下“无法获取上游表详细信息”的场景，血缘图允许展示到什么粒度。
2. 历史数据处理是否需要补数、回刷，还是仅对新数据/新查询生效。
3. 指标平台删除、下线后的同步时效与触发方式未明确。
4. 手动添加上下游血缘后，是否需要落库并长期保留，以及删除权限口径未明确。
