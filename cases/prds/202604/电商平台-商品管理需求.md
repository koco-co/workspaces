# 电商平台-商品管理需求

> 来源：内部需求文档
> 文档ID：PRD-10001
> 导入日期：2026-04-01
> 开发版本：v1.0.0

## 一、功能概述

商品管理模块是电商平台的核心功能之一，负责商品信息的全生命周期管理，包括商品创建、编辑、上下架操作及列表展示。

## 二、功能需求

### 2.1 商品列表页

**功能描述：** 展示全部商品信息，支持搜索和筛选。

**字段定义：**

| 字段名称 | 类型 | 必填 | 说明 |
|---------|------|------|------|
| 商品名称 | 文本 | 是 | 最大 100 字符 |
| 商品分类 | 枚举 | 是 | 运动鞋/休闲鞋/商务鞋 |
| SKU编号 | 文本 | 是 | 唯一标识，格式 SKU-YYYY-XXX |
| 价格 | 数字 | 是 | 单位：元，保留两位小数 |
| 库存数量 | 整数 | 是 | 最小值 0 |
| 状态 | 枚举 | 是 | 上架/下架 |
| 商品卖点 | 文本 | 否 | 最大 200 字符 |
| 创建时间 | 日期时间 | 自动 | 系统自动填充 |

**搜索筛选：**
- 支持按商品名称模糊搜索
- 支持按商品分类筛选
- 支持按状态筛选（全部/上架/下架）

**分页：** 默认每页 20 条，支持切换 10/20/50 条。

### 2.2 新增商品

**功能描述：** 通过表单新增商品。

**业务规则：**
1. 商品名称不可重复
2. SKU 编号系统自动生成，格式：`SKU-{YYYY}-{3位序号}`（如 `SKU-2026-001`）
3. 价格必须大于 0
4. 新增成功后默认状态为「下架」

### 2.3 编辑商品

**功能描述：** 支持编辑已有商品的所有字段（SKU 编号除外）。

### 2.4 上下架操作

**功能描述：** 在列表页可对单个商品执行上架/下架操作。

**业务规则：**
1. 上架：将商品状态由「下架」改为「上架」，操作后商品在前台可见
2. 下架：将商品状态由「上架」改为「下架」，操作后商品在前台隐藏
3. 库存为 0 的商品不允许上架

## 三、非功能需求

- 列表页加载时间 < 2s（1000 条数据以内）
- 新增/编辑提交响应时间 < 1s
