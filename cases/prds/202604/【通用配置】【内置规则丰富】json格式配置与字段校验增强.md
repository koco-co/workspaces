# 【通用配置】【内置规则丰富】json格式配置与字段校验增强

## 需求概述

- 本次需求覆盖蓝湖页面 **15693~15696**。
- 用户已明确依赖关系：**15696 优先**，**15693 / 15694 依赖 15696**，**15695 独立新增**。
- 核心目标分为两部分：
  - 在【通用配置】中新增并维护 json 格式配置，作为后续 json key/value 校验的基础数据源。
  - 在【数据质量】内置规则中补充 json 字段校验能力，以及字段多规则的且/或组合能力。

## 页面范围

| 页面 | 需求定位 | 与其他页面关系 |
| --- | --- | --- |
| 15696【通用配置】json格式配置 | 维护 json key 层级、中文名、value 格式、数据源类型 | 15693、15694 的基础配置来源 |
| 15693【内置规则丰富】完整性，json中key值范围校验 | 完整性校验支持 json key 范围校验 | 依赖 15696 提供 key 列表 |
| 15694【内置规则丰富】有效性，json中key对应的value值格式校验 | 有效性校验支持 json key 对应 value 格式校验 | 依赖 15696 提供 key 与 value 格式 |
| 15695【内置规则丰富】有效性，支持设置字段多规则的且或关系 | 有效性校验新增“取值范围&枚举范围”联合校验 | 独立新增，但结果查询/质量报告/规则库与同模块功能联动 |

## 依赖关系

1. **15696 → 15693 / 15694**
   - 15693 的“校验内容”选项来源于 15696 维护的 key 数据。
   - 15694 的“校验 key”与“value 格式预览”来源于 15696 维护的 key 与 value 格式。
2. **15695 独立**
   - 15695 不依赖 15696 的 json 配置数据，但共享规则配置、结果查询、质量报告、质量规则库等结果展示链路。

## 测试前置条件

- 需存在 `json` / `string` 类型字段，用于验证 15693、15694 的规则可用范围。
- 需准备 `sparkthrift2.x`、`hive2.x`、`doris3.x` 数据源场景。
- 执行 15693、15694 前，需先通过 15696 维护对应的 key、中文名称、value 格式、层级关系。
- 需准备：
  - 正常 key / value 格式数据
  - 缺失 key、非法 value 数据
  - key 数量大于 200、最好达到数千条的配置数据
- 15695 需准备可同时命中“取值范围”和“枚举值”的字段数据，以及仅命中其一的边界数据。

## 页面一：15696【通用配置】json格式配置

### 页面目标

- 新增 `json格式校验管理` 页面，支持维护 json key 配置树。
- 为后续 json key 范围校验、json value 格式校验提供基础配置数据。

### 列表与层级展示

- 支持按 **key 名称**模糊查询，且**支持查询到子层级 key 名**。
- 列表字段至少包含：
  - key
  - 中文名称
  - value 格式
  - 数据源类型
  - 创建人 / 更新人
  - 创建时间 / 更新时间
  - 操作
- 对于配置了子层级的数据展示 `+` 按钮，点击后下钻查看下一级层级；若不存在子层级，则不展示按钮。
- 展示条数按照**最外层级**统计。
- 点击 `+` 展开下一层级数据时，若识别到下一层还有数据，则继续展示 `+`。
- 最多支持配置 **5 个层级**；到第 **5 层**时，不再展示 `新增子层级` 按钮。

### 页面操作

- 单行操作：
  - `编辑`：支持编辑当前层级的 key 名称、中文名称、value 格式、数据源类型。
  - `新增子层级`：支持新增当前层级下一级的 key 名称、中文名称、value 格式。
  - `删除`：删除当前 key，若存在子层级则联动删除。
- 批量操作：
  - `批量删除`：支持勾选多行批量删除。
  - `导入`
  - `导出`
  - `新增key`

### 删除相关提示文案

- 单条删除提示：
  - `请确认是否删除key信息，若存在子层级key信息会联动删除`
- 批量删除提示：
  - `请确认是否批量删除key信息，若存在子层级key信息会联动删除`

### 新增 / 编辑 / 新增子层级字段规则

| 字段 | 必填 | 规则 |
| --- | --- | --- |
| key | 是 | 最大支持输入 255 字符；**无需进行重复性校验** |
| 中文名称 | 否 | 最大支持输入 255 字符 |
| value格式 | 否 | 最大支持输入 255 字符；支持输入正则表达式 |
| 数据源类型 | 是 | 单选；默认 `sparkthrift2.x`；可选值：`sparkthrift2.x`、`hive2.x`、`doris3.x` |

- 当配置了 `value格式` 时，支持对输入的正则内容进行测试；**仅在配置了 value 格式时才出现测试数据字段**，否则不出现。

### 验收标准

1. 可以新增、编辑、删除、批量删除 json key 配置。
2. 可以按层级新增子层级，且第 5 层不再允许新增子层级。
3. `+` 展开逻辑只在存在子层级时展示，且可继续逐层下钻。
4. 导入时必须先完成整批校验，存在错误时阻止导入并输出错误文件。
5. 导出支持当前筛选结果，文件命名符合 `json_format_+日期` 规则。

### 导入规则

- 导入文件模板名：`json_format_import_template.xlsx`
- 错误文件名：`json_format_error_20240520.xlsx`
- 点击【确定】后，系统会先进行内容校验。
- 导入校验失败提示弹窗文案：
  - `导入表格中存在错误数据，请检查后重新导入，点击导出错误文件。`
- 导入模板结构：
  - 一个 xlsx 文件包含 **5 个 Sheet**，命名依次为：`一层`、`二层`、`三层`、`四层`、`五层`
  - `一层` Sheet 字段：`key`、`中文名称`、`value格式`
  - `二层` ~ `五层` Sheet 字段：`上一层级的key名`、`key`、`中文名称`、`value格式`
- 字段必填规则：
  - `一层` Sheet 中：`key` 必填；`中文名称`、`value格式` 非必填
  - `二层` ~ `五层` Sheet 中：`上一层级的key名`、`key` 必填；`中文名称`、`value格式` 非必填
- 导入校验规则：
  1. 若 key 名称过长 / 必填项未填写 / 上一层级 key 无法找到，则对错误数据进行标红，并批注提示错误原因。
  2. **全部数据均正确后才可以导入**。
  3. 导入数据均为**新增逻辑**。
  4. 若上一层级存在重复的 key 名，默认取**第一个 key** 作为上一层级。

### 导出规则

- 导出支持对当前列表下所有数据进行导出，支持筛选导出。
- 导出文件命名规则：`json_format_+日期`，示例：`json_format_20260323`
- 导出确认提示：
  - `请确认是否导出列表数据`
- 蓝湖未明确导出文件的 Sheet 数量与字段清单。

## 页面二：15693【内置规则丰富】完整性，json中key值范围校验

### 需求目标

- 完整性校验支持对 **json 类型字段**做 **key 值范围校验**。
- 新增内置规则：`key范围校验`

### 规则配置

- 新增规则选项：`key范围校验`
- 悬浮提示文案：
  - `当选择key范围校验时，字段仅支持单选`
- 校验方法支持：
  - `包含`
  - `不包含`
- 校验内容来源：
  - 来源于 `json格式配置` 维护的数据列表。
  - 选择列表取 **key 名**。
  - 支持**多选**、**选择全部**、**输入查询**。
- 数据量边界：
  - 若 key 数据量较多，默认加载前 **200** 条展示。
  - 勾选仅对**当前层级**生效。
  - 回显内容示例：`key1-key2;key11-key22`
  - 鼠标悬浮时展示全部 key 名信息或部分，默认仅展示前两个。
  - 需考虑 **key 数量几千个** 的情况。

### 字段定义表

| 配置项 | 类型/控件 | 规则 |
| --- | --- | --- |
| 字段 | 单选字段 | 仅支持数据类型为 `json`、`string` 的字段 |
| 校验方法 | 单选 | `包含` / `不包含` |
| 校验内容 | 多选下拉 | 数据来源于 15696；支持搜索、多选、全选；默认前 200 条；仅当前层级生效 |
| 规则解释悬浮提示 | 文案提示 | `校验内容key信息需要在通用配置模块维护。` |

### 支持范围

- 支持数据源：`doris3.x`、`sparkthrift2.x`、`hive2.x`
- 仅支持数据类型为 **json、string** 的字段

### 规则说明与提示

- 规则解释：`对数据中包含的key范围校验`
- 规则描述：`校验json类型的字段中key名是否完整，对key的范围进行校验`
- 规则库悬浮提示：
  - 悬浮提示内容为规则库中的“规则解释”内容
  - `校验内容key信息需要在通用配置模块维护。`

### 结果查询 / 明细 / 日志

- 针对 **校验不通过** 的规则：
  - 支持查看明细
  - 明细标题：`查看“完整性校验-key范围校验”明细`
  - 明细记录不符合规则的数据
  - 数据列表保留全部字段
  - 校验字段标红展示
  - 下载明细数据中校验字段也标红展示
- 针对 **校验通过** 的规则：
  - 不记录明细数据
- 针对 **校验失败** 的规则：
  - 支持查看日志

### 质量报告文案

- 校验通过详情说明：
  - `符合规则key范围包含/不包含“key1-key2;key11-key22……”`
- 校验不通过原因：
  - `key范围校验未通过`
- 校验不通过详情说明：
  - `不符合规则key范围包含/不包含“key1-key2;key11-key22……”`

### 验收标准

1. 选择 `key范围校验` 后，字段控件必须退化为单选。
2. 校验内容仅能选择 15696 已维护的 key，且多层级回显符合“仅当前层级生效”规则。
3. 校验通过时不生成明细；校验不通过时可查看和下载标红明细；失败时可查看日志。
4. 质量报告中的原因与详情说明需区分“包含 / 不包含”两类口径。

## 页面三：15694【内置规则丰富】有效性，json中key对应的value值格式校验

- 位置要求：

- 新增配置项：
  - `请选择校验key`
  - `value格式预览`

  - 支持**多选**、**选择全部**、**输入查询**
  - 若 key 数据量较多，默认加载前 **200** 条展示
  - 勾选仅对**当前层级**生效

### 字段定义表

- 列表中**仅已配置 value 格式信息的 key 可以被选择**；未配置 value 格式的 key 不支持选中。

  - `校验内容为key名对应的value格式是否符合要求，value格式需要在通用配置模块维护。`

### 支持范围

- 支持数据源：`doris3.x`、`sparkthrift2.x`、`hive2.x`
- 仅支持数据类型为 **json、string** 的字段

### 规则说明

- 规则名称：`格式-json格式校验`
- 规则解释：`校验json类型的字段中key对应的value值是否符合规范要求`
- 规则分类：有效性校验
- 关联范围：字段
- 规则库悬浮提示内容为规则库中的“规则解释”内容

  - 数据列表保留全部字段
  - 校验字段标红展示
  - 下载明细数据中校验字段也标红展示
- 针对 **校验通过** 的规则：
  - 不记录明细数据
- 针对 **校验失败** 的规则：
  - 支持查看日志

### 质量报告文案

- 校验通过详情说明：
  - `符合规则key为“key1-key2;key11-key22…”时的value格式要求`
- 校验不通过原因：
  - `key对应value格式校验未通过`
- 校验不通过详情说明：
  - `不符合规则key为“key1-key2;key11-key22…”时的value格式要求`

### 验收标准

1. 仅已配置 value 格式的 key 可被选中参与校验。
2. `value格式预览` 只能展示勾选 key 对应的数据，且需支持分页。
3. 校验通过时不记录明细；校验不通过时详情、下载和页面标红一致；失败时支持查看日志。
4. 规则入口位置必须位于自定义正则上方。

## 页面四：15695【内置规则丰富】有效性，支持设置字段多规则的且或关系

### 需求目标

- 有效性校验支持对**取值范围**和**枚举值**进行联合判断。
- 新增规则名称：`取值范围&枚举范围`

### 规则配置结构

- 第一行：`取值范围设置`
  - 逻辑与原有取值范围规则保持一致
- 第二行：`枚举值设置`
  - 逻辑与原有枚举值规则保持一致
  - 支持选择 `in / not in`
- 第三行：`取值范围和枚举值关系`
  - 支持配置 `且 / 或`
  - 支持配置过滤条件

### 字段定义表

| 配置项 | 类型/控件 | 规则 |
| --- | --- | --- |
| 取值范围设置 | 组合条件 | 与原取值范围规则一致 |
| 枚举值设置 | 组合条件 | 与原枚举值规则一致；支持 `in / not in` |
| 取值范围和枚举值关系 | 单选 | `且` / `或` |
| 过滤条件 | 条件输入 | 与组合规则一起保存、生效 |

### 字段与类型支持

- 新增规则支持：
  - `string`
  - 数值类型字段
- `string` 类型默认强转为 `double`

### 兼容影响

- 原来的枚举值配置规则也同步新增该交互能力。

### 规则库信息

- 规则名称：`取值范围&枚举范围`
- 规则解释：`取值范围和枚举范围的联合校验`
- 规则分类：有效性校验
- 关联范围：字段
- 规则描述：`校验字段值取值范围和枚举范围是否符合要求，支持配置规则且或关系`

- 针对 **校验不通过** 的规则：

- 针对 **校验失败** 的规则：
  - 支持查看日志

### 质量报告文案

- 联合规则校验通过详情说明示例：
  - `符合规则“取值范围>1”且“枚举值in ‘1,2,3’”`
- 联合规则校验不通过原因：
  - `不符合有效性规则`
- 联合规则校验不通过详情说明示例：
  - `不符合规则“取值范围>1”且“枚举值in ‘1,2,3’”`

### 验收标准

1. 第一行、第二行、第三行配置分别保存，执行结果与组合关系一致。
2. `且` 模式下需同时满足取值范围和枚举值规则，`或` 模式下任一满足即可。
3. 枚举值规则支持 `in / not in`，且质量报告中同步体现 not in 场景。
4. `string` 字段按 `double` 强转后再参与联合计算。
