# 【内置规则丰富】完整性-json中key值范围校验

## 需求概述

- 完整性校验新增 `key范围校验`。
- 用于校验 json 类型字段中 key 名是否符合预设范围。
- 本需求依赖 15696 提供的 json key 配置数据。

## 依赖关系

- 依赖 `【通用配置】json格式配置`：
  - 校验内容来源于 15696 维护的 key 数据。
  - 需支持千级 key、多层级配置和仅当前层级生效。

## 规则配置

- 新增规则：`key范围校验`
- 悬浮提示：
  - `当选择key范围校验时，字段仅支持单选`
- 校验方法：
  - `包含`
  - `不包含`

| 配置项 | 规则 |
| --- | --- |
| 字段 | 仅支持 `json`、`string` 字段，且字段仅支持单选 |
| 校验内容 | 来源于 15696 维护的 key 列表；支持搜索、多选、全选 |
| 数据量边界 | 默认展示前 200 条；需考虑千级 key 场景 |
| 层级规则 | 勾选仅对当前层级生效；回显示例 `key1-key2;key11-key22` |
| 支持数据源 | `doris3.x` / `sparkthrift2.x` / `hive2.x` |

## 规则集管理与提示

- 规则解释：`对数据中包含的key范围校验`
- 规则描述：`校验json类型的字段中key名是否完整，对key的范围进行校验`
- 悬浮提示：
  - `校验内容key信息需要在通用配置模块维护。`

## 结果查询 / 质量报告 / 日志

- 校验不通过：
  - 支持查看明细
  - 明细标题：`查看“完整性校验-key范围校验”明细`
  - 明细保留全部字段，校验字段标红
  - 下载明细中校验字段也标红
- 校验通过：
  - 不记录明细数据
- 校验失败：
  - 支持查看日志

### 质量报告文案

- 通过：
  - `符合规则key范围包含/不包含“key1-key2;key11-key22……”`
- 不通过原因：
  - `key范围校验未通过`
- 不通过详情：
  - `不符合规则key范围包含/不包含“key1-key2;key11-key22……”`

## 配套任务场景

- 支持先在 `规则集管理` 中创建包含 `key范围校验` 的规则包，再在 `规则任务管理` 中创建规则任务并绑定 / 管理该规则包。
- 支持抽样场景、分区场景下配置并运行 `key范围校验`。
- 当 key 配置已关联质量任务后，在 15696 页面删除 key，历史规则仍需正常回显，质量任务仍需正常运行。
- 用户补充要求：脏数据库变更后，脏数据存储功能仍需正常。

## 重点测试关注点

1. 千级 key、多层级、前 200 条展示、仅当前层级生效的组合场景。
2. 通过 / 不通过 / 失败三条结果路径的详情、日志、下载标红。
3. 抽样、分区、`规则集管理 → 规则任务管理` 绑定链路，以及历史 key 被删除后的回归链路。
