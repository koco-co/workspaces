# 【内置规则丰富】有效性-支持设置字段多规则的且或关系

## 需求概述

- 有效性校验新增联合规则：`取值范围&枚举范围`
- 支持将取值范围和枚举值两类规则进行联合判断。
- 支持 `且 / 或` 关系，以及 `in / not in` 枚举模式。

## 规则配置结构

| 行 | 配置项 | 规则 |
| --- | --- | --- |
| 第一行 | 取值范围设置 | 与原有取值范围规则保持一致 |
| 第二行 | 枚举值设置 | 与原有枚举值规则保持一致；支持 `in / not in` |
| 第三行 | 取值范围和枚举值关系 | 支持 `且 / 或`，支持过滤条件 |

## 字段与类型支持

- 支持字段类型：
  - `string`
  - 数值类型
- `string` 类型默认强转为 `double`

## 规则库信息

- 规则名称：`取值范围&枚举范围`
- 规则解释：`取值范围和枚举范围的联合校验`
- 规则分类：有效性校验
- 关联范围：字段
- 规则描述：`校验字段值取值范围和枚举范围是否符合要求，支持配置规则且或关系`

## 结果查询 / 质量报告 / 日志

- 校验不通过：
  - 支持查看明细
  - 明细标题：`查看“有效性校验”明细`
  - 明细保留全部字段，校验字段标红
  - 下载明细中校验字段也标红
- 校验通过：
  - 不记录明细数据
- 校验失败：
  - 支持查看日志

### 质量报告文案

- 通过示例：
  - `符合规则“取值范围>1”且“枚举值in ‘1,2,3’”`
- 不通过原因：
  - `不符合有效性规则`
- 不通过示例：
  - `不符合规则“取值范围>1”且“枚举值in ‘1,2,3’”`
- 现有枚举值规则额外要求：
  - 质量报告详情说明新增 `not in / 不包含` 场景

## 历史兼容要求

- 用户明确要求：
  - **历史规则 / 历史任务 / 历史报告不支持回显枚举值配置**
  - **只有新增的数据才会带出枚举值配置**

## 配套任务场景

- 支持抽样场景、分区场景下配置并运行联合规则。
- 支持在规则包中配置联合规则，再由新建规则任务绑定规则包。

## 重点测试关注点

1. `且 / 或` 两种关系的执行差异。
2. `in / not in` 的配置、运行和质量报告说明差异。
3. `string` 强转 `double` 的联合计算逻辑。
4. 历史规则 / 历史任务 / 历史报告与新增数据的兼容差异。

## 非阻断待确认事项

1. 仅配置取值范围、仅配置枚举值、过滤条件为空时的执行口径未明确。
