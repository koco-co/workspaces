# 15525【内置规则丰富】一致性校验：多表数据一致性比对

## 一、功能概述

**功能目标：** 在数据质量模块的规则配置中，新增「一致性校验」规则类型，内置「多表数据一致性比对」规则。用户可以配置 2-10 张数据表，指定主键和校验字段，系统在任务执行时判断多表之间的数据是否一致，并在质量报告中展示校验结果及明细。

**适用用户角色：**

| 角色 | 权限说明 |
|------|----------|
| 数据质量配置人员 | 可在规则配置中新增/编辑一致性校验规则 |
| 数据质量查看人员 | 可查看质量报告和检验明细 |

---

## 二、页面结构与入口

**页面入口路径：** 数据质量 → 规则配置 → 点击【新增规则】→ 弹窗中选择「校验类型：一致性校验」→ 切换到「多表数据一致性比对」配置面板

**页面布局说明：** 弹窗式配置页，顶部选择校验类型（一致性校验），下方展开「多表数据一致性比对设置」，包含对比表配置区域 + 比对字段设置区域（条件显示）；底部有【确定】【取 消】两个操作按钮。

### 5.2 业务逻辑规则

| 规则编号 | 规则描述 |
|----------|----------|
| B-001 | 最多添加 10 个对比表（含对比表1和对比表2，合计最多10个） |
| B-002 | 主键支持多选，多选后按照联合主键判断数据一致性 |
| B-003 | 若不勾选「选择校验字段」，则只按照主键校验数据是否存在差异，只要存在差异即判为校验不通过 |
| B-004 | 针对「校验通过」的规则，不记录明细数据 |
| B-005 | 针对「校验不通过」的规则，支持查看明细，明细分为「逻辑主键匹配但数据不匹配」和「逻辑主键不匹配」两种情况 |
| B-006 | 针对「校验失败」的规则（系统执行异常），支持查看日志 |
| B-007 | 明细数据仅存储校验主键和校验字段，不做标红处理 |
| B-008 | 质量报告中不一致的数据条数记录在「详情说明」列，格式：「不一致的数据共xx条」 |

## 六、流程与状态机

### 6.1 配置规则流程

```
进入规则配置页
  → 点击【新增规则】（弹窗）
  → 选择「校验类型」= 一致性校验
  → 展开「多表数据一致性比对设置」
  → 配置对比表1（表名 + 主键）
  → 配置校验表主键
  → 配置对比表2（表名 + 主键）
  → （可选）添加更多对比表（最多10个）
  → （可选）选择「校验字段」→ 展示「比对字段设置」
  → 点击【确定】→ 校验通过 → 弹窗关闭，规则保存至列表
               → 校验不通过 → 高亮必填项，弹窗留存
  → 点击【取 消】→ 弹窗关闭，不保存
```

### 6.2 任务执行后质量报告状态

```
规则执行结果
  → 校验通过 → 质量报告显示「校验通过」，不记录明细
  → 校验不通过 → 质量报告显示「校验不通过」，支持【查看详情】
  → 校验失败（系统异常）→ 质量报告显示「校验失败」，支持查看日志
```
