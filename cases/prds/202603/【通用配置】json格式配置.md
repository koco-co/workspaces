## 功能概述

在「数据质量 → 通用配置」菜单下新增「JSON格式配置」子页面。

**功能作用**：统一管理 JSON 字段的 key 结构和 value 格式校验规则（正则表达式），列表以「数据源类型」列区分每条配置所属的数据源类型（`sparkthrift2.x` / `hive2.x` / `doris3.x`），供数据质量规则引用。

**菜单层级**：

```
数据质量
  └── 通用配置
        ├── 维度配置（已有功能）
        └── JSON格式配置（本需求，新增）   ← 前端路由：views/valid/generalConfig/jsonFormat
```

> 源码说明：以下设计基于 PRD 和需求澄清形成的规范，前端实现应严格遵循该规范。

---

## 前置条件

- 用户已登录系统，具备「通用配置」菜单的访问权限
- **无数据库依赖**：纯配置管理功能，不需要准备测试数据源或建表 SQL
- 列表「数据源类型」列支持按 `sparkthrift2.x`、`hive2.x`、`doris3.x` 下拉筛选

## 页面详细设计

### 4.1 JSON格式配置 - 列表页

**菜单入口**：数据质量 → 通用配置 → JSON格式校验管理

#### 4.1.1 页面顶部功能区

| 功能组件 | 类型 | 说明 |
|---------|------|------|
| 搜索框 | 文本输入 | 按 key 名称模糊搜索，可查询到任意层级子层级的 key |
| 【新增】 | 操作按钮 | 新增根级 key，打开新增弹窗 |
| 【导入】 | 操作按钮 | 打开文件选择器，选择 Excel 文件（.xlsx / .xls）导入 |
| 【导出】 | 操作按钮 | 弹出确认弹窗后下载 Excel |
| 【批量删除】 | 操作按钮 | 勾选行后激活，弹出确认弹窗后批量删除 |

#### 4.1.2 列表字段定义

| 列名 | 类型 | 说明 | 可为空 |
|------|------|------|--------|
| 复选框 | checkbox | 用于批量选择，首列 | — |
| key | 文本（树形） | JSON 字段名，树形展示，支持展开/折叠子层级 | 否 |
| 数据源类型 | 文本（带列头下拉筛选） | 枚举：`sparkthrift2.x` / `hive2.x` / `doris3.x`；列名右侧有下拉筛选图标，点击可按数据源类型过滤列表；默认不筛选，展示全部数据源的配置 | 否 |
| 中文名称 | 文本 | key 的中文描述 | 是 |
| value格式 | 文本 | 正则表达式格式校验规则 | 是 |
| 创建时间 | 时间 | 格式：YYYY-MM-DD HH:mm:ss | — |
| 更新时间 | 时间 | 格式：YYYY-MM-DD HH:mm:ss | — |
| 创建人 | 文本 | — | — |
| 更新人 | 文本 | — | — |
| 操作 | 操作列 | 见 4.1.3 | — |

#### 4.1.3 操作列规则（树形节点类型决定）

| 节点类型 | 操作按钮 |
|---------|---------|
| 层级 1-4 的任意节点（无论是否有子层级） | 【编辑】 / 【新增子层级】 / 【删除】 |
| 第 5 层节点（最深层，用户澄清确认） | 【编辑】 / 【删除】（**无**【新增子层级】） |

> 业务规则：PRD 原文描述叶子节点无「新增子层级」；用户澄清确认最大深度为 5 层，第 5 层强制不显示【新增子层级】。

#### 4.1.4 树形层级规则

- 最大支持 **5 层**嵌套（根级为第 1 层，最深第 5 层）
- 默认展示状态：根级展开，子层级手动展开/折叠

#### 4.1.5 搜索交互规则

- 按 key 名称模糊搜索，可查询到子层级 key
- 搜索时列表只展示匹配项及其父级路径

### 4.2 JSON格式配置 - 新增 / 编辑弹窗

**入口**：列表页【新增】按钮（根级）/ 操作列【新增子层级】/ 操作列【编辑】

#### 4.2.1 表单字段定义

| 字段名 | 必填 | 类型 | 校验规则 | 默认值 | 备注 |
|--------|------|------|---------|--------|------|
| 数据源类型 | **必填** | 单选下拉 | 不可为空 | sparkthrift2.x | 新增时可选；编辑时可修改；枚举：sparkthrift2.x / hive2.x / doris3.x |
| key | **必填** | 文本输入 | 最大 255 字符；不做唯一性校验 | — | 同一数据源类型下允许重复（用户澄清确认） |
| 中文名称 | 非必填 | 文本输入 | 最大 255 字符 | — | — |
| value格式 | 非必填 | 文本输入（正则） | 最大 255 字符 | — | 填写后联动显示「测试数据」字段 |
| 测试数据 | 条件显示 | 文本输入 | — | — | 仅当「value格式」已填写时显示；输入后实时显示正则匹配结果 |

#### 4.2.2 字段交互规则

**value格式 → 测试数据 联动规则**：
- 「value格式」字段为空 → 「测试数据」字段**不显示**
- 「value格式」字段有值 → 「测试数据」字段显示
- 用户在「测试数据」输入框输入内容 → 系统**实时**用正则规则匹配
  - 匹配成功：输入框旁显示绿色「匹配成功」文字
  - 匹配失败：输入框旁显示红色「不匹配」文字

**字段超长错误提示**：
- key / 中文名称 / value格式 超过 255 字符 → 输入框下方红色提示「最多255个字符」

**key 必填校验**：
- key 为空提交 → 必填提示「请输入key」

**数据源类型默认值**：
- 新增时默认选中 `sparkthrift2.x`

#### 4.2.3 提交结果

| 场景 | 结果 |
|------|------|
| 新增/编辑成功 | 弹窗关闭 + Toast 提示「操作成功」+ 列表自动刷新 |
| 必填项为空 | 弹窗不关闭，输入框下方显示红色必填提示 |
| 字段超 255 字符 | 弹窗不关闭，输入框下方显示「最多255个字符」 |

### 4.3 JSON格式配置 - 删除操作

#### 4.3.1 单条删除

- 点击操作列【删除】按钮
- 弹出确认弹窗，文案：「确认删除该key信息？」
- 若该 key 有子层级：确认后**级联删除**所有子层级 key

#### 4.3.2 批量删除

- 勾选列表中一条或多条数据
- 点击顶部【批量删除】按钮
- 弹出确认弹窗，文案：`"请确认是否批量删除key信息，若存在子层级key信息会联动删除"`（PRD 原文）
- 点击确认 → 执行删除

---

### 4.4 JSON格式配置 - 导出

- 点击【导出】按钮
- 弹出确认弹窗，文案：`"请确认是否导出列表数据"`（PRD 原文）
- 确认 → 下载 Excel 文件
- **文件命名规则**：`json_format_YYYYMMDD`（如 `json_format_20260323`）
- **支持筛选导出**：按当前数据源类型列过滤器和搜索条件筛选后导出

---

### 4.5 JSON格式配置 - 导入

- 点击【导入】按钮 → 打开文件选择器
- **支持格式**：Excel（.xlsx / .xls）
- **导入逻辑**：所有数据按**新增**处理，不做覆盖更新
- **异常情况**：非 Excel 格式 → 给出格式错误提示
