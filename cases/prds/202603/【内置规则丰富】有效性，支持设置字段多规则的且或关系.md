# [15695] 【内置规则丰富】有效性，支持设置字段多规则的且或关系

## 1. 基本信息

- 模块: `data-assets` / 数据质量
- 需求版本: `v6.4.10`
- 开发版本: `6.3岚图定制化分支`
- 分支上下文:
  - backend: `.repos/CustomItem/dt-center-assets` @ `release_6.3.x_ltqc`
  - frontend: `.repos/CustomItem/dt-insight-studio` @ `dataAssets/release_6.3.x_ltqc`
- 关联能力:
  - 现有 `取值范围` 规则
  - 现有 `枚举值` 规则
  - 新增 `取值范围&枚举范围` 联合规则

## 2. 需求背景

现有有效性校验已经分别支持“取值范围”和“枚举值”两种单规则配置，但业务在校验字段合法性时，往往需要同时满足区间约束与枚举约束。本需求要求新增“取值范围&枚举范围”联合规则，并支持设置“取值范围”和“枚举值”之间的 `且/或` 关系，同时将原有枚举值规则补齐 `not in` 配置与报告文案。

结合蓝湖原型文本、用户补充截图和现有源码，可以确认本需求仍然落在【规则任务管理】的现有配置框架内，重点是扩展规则行结构、关系表达能力和报告详情说明，而不是新建独立业务页。

## 3. 变更范围

| 类别 | 影响对象 | 变更说明 |
|------|----------|----------|
| 内置规则 | 规则库配置 | 新增 `取值范围&枚举范围` 规则条目 |
| 规则配置 | 规则任务管理编辑页 | 支持第一行配置取值范围、第二行配置枚举值、第三行配置两者关系与过滤条件 |
| 枚举值规则回归 | 原枚举值规则 | 枚举值配置同步新增 `not in` 选项 |
| 字段类型 | string / 数值字段 | 新规则同时支持 string 和数值字段；string 类型默认强转为 double 参与判断 |
| 质量结果 | 数据质量报告 | 对联合规则输出“符合/不符合规则‘取值范围…’且/或‘枚举值…’”说明 |
| 失败追踪 | 查看详情 / 查看日志 | 失败时支持查看详情与日志，详情中目标字段高亮 |

## 4. 页面与交互详细设计

### 4.1 规则库配置

- 菜单路径: `数据质量 → 规则库配置`
- 新增规则名称: `取值范围&枚举范围`
- 规则解释: `取值范围和枚举范围的联合校验`
- 规则分类: `有效性校验`
- 关联范围: `字段`
- 规则描述: `校验字段值取值范围和枚举范围是否符合要求，支持配置规则且或关系`

### 4.2 规则任务管理 - 新建/编辑单表校验规则页

- 菜单路径: `数据质量 → 规则任务管理`
- 页面仍使用三步向导: `监控对象 → 监控规则 → 调度属性`
- 在 `监控规则` 步骤新增 `取值范围&枚举范围` 规则配置方式

#### 4.2.1 配置结构

| 行号 | 配置内容 | 说明 |
|------|----------|------|
| 第一行 | 取值范围设置 | 逻辑与原有取值范围规则保持一致，支持双操作符与双阈值 |
| 第二行 | 枚举值设置 | 逻辑与原有枚举值规则保持一致，支持 `in / not in` |
| 第三行 | 取值范围和枚举值关系 + 过滤条件 | 新增 `且 / 或` 切换，用于决定联合判断方式 |

#### 4.2.2 交互规则

- 联合规则支持 `string`、数值类型字段判断
- 当字段类型为 `string` 时，默认按 `double` 逻辑参与数值判断
- `取值范围和枚举值关系` 支持 `且` 与 `或`
- 原来的枚举值规则也同步新增 `not in` 选项
- 蓝湖补充截图进一步确认:
  - 联合规则的三行配置位于同一规则块中
  - 枚举值配置通过“点击配置”进入具体枚举值选择
  - 关系行除 `且/或` 外仍保留过滤条件配置位

### 4.3 数据质量报告与失败追踪

- 菜单路径: `数据质量 → 数据质量报告`
- 联合规则详情展示要求:

| 场景 | 质检结果 | 未通过原因 | 详情说明 | 操作 |
|------|----------|------------|----------|------|
| 通过 | 校验通过 | `--` | `符合规则“取值范围>1”且/或“枚举值in 1,2,3”` | 不展示查看详情 |
| 失败 | 校验不通过 | `不符合有效性规则` | `不符合规则“取值范围>1”且/或“枚举值in 1,2,3”` | 展示查看详情；失败时支持查看日志 |

- 原枚举值规则的质量报告详情说明需要补齐 `not in` 场景文案
- 失败明细记录不符合规则的数值，数据列表保留全部字段，校验字段标红展示，下载明细数据中目标字段保持同样高亮语义

## 5. 源码补充事实

- 前端 `rule/index.tsx` 已检出 `range.firstOperator`、`range.firstThreshold`、`range.condition`、`range.secondOperator`、`range.secondThreshold` 等字段
- 后端 `Range.java` 已明确存在 `condition / firstOperator / firstThreshold / secondOperator / secondThreshold` 字段，可承载联合规则的且或关系
- 当前需求的专属文案 `取值范围&枚举范围` 仍主要来自 PRD 与截图，表明本分支更接近“通用框架已就绪，专属规则项需补齐”状态
