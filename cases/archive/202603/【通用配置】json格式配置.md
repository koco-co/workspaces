### 列表页

#### 列表默认展示

##### 【P0】验证列表默认展示完整功能流程

> 前置条件
```
用户已登录系统，具备「通用配置」菜单的访问权限，JSON格式配置页面功能已开启；sparkthrift2.x数据源下已存在多层级数据（根级key "vehicleInfo"（车辆信息，value格式："^[A-Z]{2}\d{5}$"）下有子级key "plateNo"（车牌号），"plateNo"下有孙级key "plateType"（车牌类型）），且sparkthrift2.x根级key总数超过一页（至少11条根级key）。
```

| 编号 | 步骤 | 预期 |
| --- | --- | --- |

| 2 | 查看列表列字段 | 列表共显示以下列：复选框（首列）、key、数据源类型（列名右侧有下拉筛选图标）、中文名称、value格式、创建时间（格式：YYYY-MM-DD HH:mm:ss）、更新时间（格式：YYYY-MM-DD HH:mm:ss）、创建人、更新人、操作（含【编辑】/【新增子层级】/【删除】按钮）；已存在的记录行 key 列显示"vehicleInfo"、数据源类型列显示"sparkthrift2.x"、中文名称列显示"车辆信息"、value格式列显示"^[A-Z]{2}\d{5}$" |
| 3 | 观察列表默认展示状态 | 根级 key "vehicleInfo" 默认展开，显示其子级 key "plateNo"；"plateNo" 的子级 "plateType" 默认折叠，不显示 |
| 4 | 查看列表底部分页控件 | 列表底部显示分页控件，包含当前页码、总条数、每页条数选择器、上一页/下一页按钮 |

| 4 | 勾选 "vehicleInfo"，点击【批量删除】，在确认弹窗中点击【确认】 | 弹窗关闭，Toast 提示「操作成功」，列表刷新，"vehicleInfo"、"plateNo"、"plateType" 均从列表消失 |
