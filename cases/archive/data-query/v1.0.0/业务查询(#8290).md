> 用例步骤

| 编号 | 步骤 | 预期 |

| 1 | 进入【业务查询-SQL查询】页面，【数据源详情】中已与数据库账号关联的用户登录，进入【业务查询-SQL查询】页面，查看「数据库导航」数据 | 「数据库导航」展示所有已连接数doris据源，以树形图展示，一级节点为数据源连接名称（显示格式“${doris数据源icon}${数据源连接名称}”），二级节点为catalog名称（显示格式“${catalog icon}${catalog名称}”），三级节点为schema名称（显示格式“${schema icon}${schema名称}”），四级节点为table名称（显示格式“${table icon}${table名称}”） |
| 2 | 查看一级节点 | 一级节点展示当前账号有权限的所有的已连接数据源（不同数据源连接但数据源信息相同，展示为多个一级节点） |
| 3 | 查看二级节点 | 二级节点展示当前账号有权限的已连接数据源下的所有catalog：“internal”+${外部数据目录名称} |
| 4 | 查看三级节点 | 三级节点展示当前账号有权限的已连接数据源对应catalog下所有schema |
| 5 | 查看四级节点 | 四级节点展示当前账号有权限的已连接数据源对应catalog下对应schema下所有table |

| 编号 | 步骤 | 预期 |
| --- | --- | --- |
| 1 | 进入【业务查询-SQL查询】页面，查看搜索框置灰提示 | 搜索框置灰提示：请输入名称 |
| 2 | 搜索框输入任意值“A”，回车或点击“查询icon” | 「数据库导航」展示“数据源名称”或“catalog”或“schema”或“table”包含“A”的数据库信息(已展开的数据)，以树形图展示，一级节点为数据源连接名称（显示格式“${doris数据源icon}${数据源连接名称}”），二级节点为catalog名称（显示格式“${catalog icon}${catalog名称}”），三级节点为schema名称（显示格式“${schema icon}${schema名称}”），四级节点为table名称（显示格式“${table icon}${table名称}”） |
| 3 | 清空搜索框，回车或点击“查询icon” | 「数据库导航」展示所有已连接的数据库信息，以树形图展示，一级节点为数据源连接名称（显示格式“${doris数据源icon}${数据源连接名称}”），二级节点为catalog名称（显示格式“${catalog icon}${catalog名称}”），三级节点为schema名称（显示格式“${schema icon}${schema名称}”），四级节点为table名称（显示格式“${table icon}${table名称}”） |
