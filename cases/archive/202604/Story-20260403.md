5、场景口径：
- `AND_PASS`：用于验证 `score_num >= 80` 且 `score_num in (88,95)` 时整体通过。
- `OR_PASS`：用于验证 `score_num >= 80` 或 `score_num in (60,75)` 时整体通过。
- `FAIL_DETAIL`：用于验证联合规则失败时仅记录 `record_id` 为 `6`、`7` 的失败数据，`record_id` 为 `8` 的通过数据不进入明细。
- `FILTER_SCOPE`：用于验证过滤条件 `scenario_code = 'FILTER_SCOPE' AND region_code = '过滤内'` 仅校验 `record_id` 为 `9`、`10` 的数据。
- `STRING_PASS`：用于验证 `score_text` 为 string 字段时按 double 强转后参与 `>= 80` 判断。
- `NOT_IN_PASS`：用于验证联合规则 `score_num >= 80` 且 `score_num not in (81,82)` 的通过场景。
- `ENUM_IN_PASS` / `ENUM_IN_FAIL`：用于验证原有枚举值规则 `in (88,95)` 的通过与失败场景。
- `ENUM_NOTIN_PASS` / `ENUM_NOTIN_FAIL`：用于验证原有枚举值规则 `not in (81,82)` 的通过与失败场景。
