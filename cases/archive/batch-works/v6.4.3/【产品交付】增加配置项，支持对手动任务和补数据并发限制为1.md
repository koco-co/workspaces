## 增加配置项，支持对手动任务和补数据并发限制为1

##### 【P1】验证补数据任务限制为1时并发控制生效

> 前置条件
```
存在五个周期任务A~E
任务A:
select sleep(100);
SELECT 1;

任务B:
SELECT 1;

任务C:
SELECT 1;

任务D:
SELECT 1;

任务E:
SELECT 1;
```

| 5 | 保存并提交任务A~E | 成功提交至运维中心 |

##### 【P2】验证项目管理页新增「手动任务/补数据实例默认并发数」配置项展示

> 前置条件
```
无
```

> 用例步骤

| 编号 | 步骤 | 预期 |
| --- | --- | --- |
| 1 | 进入【项目管理】-【操作设置】页面 | 成功进入目标页面，页面内容正常加载显示，无报错 |
| 2 | 检查配置项列表 | 新增配置项“手动任务/补数据实例默认并发数”显示在操作设置区域 |
| 3 | 查看悬停问号图标提示信息 | 限制运行的最大实例数，该配置适用于手动任务和补数据任务选择 【不限制】 时，系统将在资源允许的情况下，将同一批次内的所有实例同时提交至 Yarn 运行；选择 【限制】 时，可设置单个批次内允许同时运行的最大实例数，超出部分将排队等待资源释放后再运行。 |
| 4 | 点击“编辑”按钮 | 弹出“任务实例默认并发数配置”弹窗, 分别展示手动任务和补数据任务的并行实例数配置 |
| 5 | 检查默认状态 | 均默认展示为【不限制】并显示“编辑”按钮 |
