##### 【P2】验证项目管理页新增配置项展示

> 用例步骤

| 编号 | 步骤 | 预期 |
| --- | --- | --- |
| 1 | 进入【项目管理】-【操作设置】页面, 检查配置项列表 | 新增配置项“手动任务/补数据实例默认并发数”显示在操作设置区域 |
| 2 | 查看悬停问号图标提示信息 | 限制运行的最大实例数，该配置适用于手动任务和补数据任务<br>选择 「不限制」 时，系统将在资源允许的情况下，将同一批次内的所有实例同时提交至 Yarn 运行；<br>选择 「限制」 时，可设置单个批次内允许同时运行的最大实例数，超出部分将排队等待资源释放后再运行。 |
| 3 | 点击“编辑”按钮 | 弹出“任务实例默认并发数配置”弹窗, 分别展示手动任务和补数据任务的并行实例数配置 |
| 4 | 检查默认状态 | 均默认展示为「不限制」并显示“编辑”按钮 |

##### 【P1】验证手动任务限制为1时并发控制生效

> 前置条件

```
存在五个手动任务taskA~taskE, 且taskA~taskD任务均依赖于任务taskE
任务taskA:
select sleep(10);
SELECT 1;

任务taskB:
select sleep(10);

任务taskC:
select sleep(10);

任务taskD:
select sleep(10);

任务taskE:
select sleep(10);
```

> 用例步骤

| 编号 | 步骤 | 预期 |
| --- | --- | --- |
| 1 | 进入【项目管理】-【操作设置】, 将手动任务的默认并发数设置为“限制 1” | 配置成功保存 |
| 2 | 进入「数据开发-手动任务」, 保存并提交mysql任务taskA~taskE (任务taskA~taskD依赖于任务taskE) | 成功提交至运维中心 |
| 3 | 进入「运维中心-手动任务管理」, 运行任务taskE(包含下游任务taskA~taskD), 其他配置项保持默认后确定 | 「手动任务实例」中新增实例a (该实例包含taskA~taskE) |
| 4 | 进入「手动任务-手动任务实例」, 实例a详情页面 | 任务taskE执行完毕后, 任务taskA~taskD按照并发为1的规则依次执行, 未执行的任务状态处于「等待运行」 |
| 5 | 返回「手动任务-手动任务管理」页面, 重新运行taskE(包含下游任务taskA~taskD, 且「最大并行实例数」为不限制) | 「手动任务实例」中新增实例b |
| 6 | 进入「手动任务-手动任务实例」, 手动任务实例b详情页面 | 任务taskE执行完毕后, 任务taskA~taskD按照并发为「不限制」的规则执行, 任务taskA~taskD立马进入到「运行中」状态 |
