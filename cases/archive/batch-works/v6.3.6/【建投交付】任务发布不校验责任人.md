## 离线开发-项目管理-发布管理

##### 验证任务发布至目标项目功能正常_目标项目责任人不存在 「P1」

> 前置条件
```
1）已存在项目A、B、C、，项目B为项目A所绑定的目标项目（即项目A为测试项目、项目B为生产项目、项目C为导入发布包使用的项目）
2）项目A中已经添加成员：用户A、用户B
3）项目B中已经添加成员：用户A
4）项目C中已经添加成员：用户A
```

| 编号 | 步骤 | 预期 |
| --- | --- | --- |
| 1 | 进入项目A的【数据开发】页面，新建周期任务：sql任务A，设置【基础属性-责任人】为用户B，保存并提交 | 1）sql任务A修改责任人成功，责任人显示为修改后的用户 |
| 2 | 进入项目A的【数据开发-发布任务】页面，将sql任务A打包，导出发布包A | 2）sql任务A保存并提交成功，系统给出提交成功提示，状态更新为下一处理阶段 |
| 3 | 用户A进入项目C的【数据开发-发布任务】页面，将发布包A再次导入当前项目（项目C） | 发布包A导出成功，系统给出成功反馈，相关页面/数据状态更新为最新 |
| 4 | 用户A进入项目A的【数据开发-发布任务】页面，将sql任务A打包，一键发布至项目B | 1）项目C导入发布包成功，sql任务A责任人显示为：用户A |
| 5 | 用户B进入项目A的【数据开发-发布任务】页面，将sql任务A打包，一键发布至项目B | 2）sql任务A临时运行成功，补数据运行成功，状态更新为【成功】，运行/执行结果符合预期，日志无报错 |
| 6 |  | 一键发布至项目B失败，提示目标项目责任人不存在 |
| 7 |  | 一键发布至项目B失败，提示目标项目责任人不存在 |
