## 补数据实例日志打印

##### 【P2】验证失败日志时间戳正常打印

> 前置条件
```
sql语句列表：
ddl：
-- 创建表（带 COMMENT 和 TBLPROPERTIES）
CREATE TABLE IF NOT EXISTS sales_orders (
order_id     STRING COMMENT '订单ID',
customer_id  STRING COMMENT '客户ID',
order_date   DATE   COMMENT '订单日期',
amount       DECIMAL(10,2) COMMENT '订单金额'
)
USING parquet
PARTITIONED BY (order_date)
TBLPROPERTIES ('comment' = '销售订单表');
-- 修改表结构（新增字段）
ALTER TABLE sales_orders
ADD COLUMNS (status STRING COMMENT '订单状态');
-- 删除表
DROP TABLE IF EXISTS sales_orders;
dql：
-- 简单查询
SELECT * FROM sales_orders WHERE amount > 100;
-- 窗口函数
SELECT customer_id,
SUM(amount) OVER (PARTITION BY customer_id ORDER BY order_date) AS running_total
FROM sales_orders;
-- GROUPING SETS（Spark 3.5 增强）
SELECT customer_id, order_date, SUM(amount) AS total
FROM sales_orders
GROUP BY GROUPING SETS ((customer_id), (order_date));
dml：
-- 插入数据
INSERT INTO sales_orders
VALUES ('O1001', 'C001', DATE'2025-01-01', 199.99, 'PAID');
-- 覆盖插入
INSERT OVERWRITE sales_orders
VALUES ('O1002', 'C002', DATE'2025-01-02', 299.50, 'PENDING');
-- 更新数据（Spark 3.1 引入 Delta 支持 UPDATE，3.5 中已稳定）
UPDATE sales_orders
SET status = 'CANCELLED'
WHERE order_id = 'O1002';
-- 删除数据
DELETE FROM sales_orders
WHERE amount < 100;
3.5新增sql：
SELECT order_id, order_date + INTERVAL '2' DAY AS new_date
FROM sales_orders;
```

> 用例步骤

| 编号 | 步骤 | 预期 |
| --- | --- | --- |
| 1 | 进入【数据开发模块】页面，点击新建oushudb sql周期任务 | 系统提示新建成功，列表顶部出现新建的记录，记录内容与填写一致 |
| 2 | 输入运行失败的sql，点击保存、提交 | 任务保存、提交完成 |
| 3 | 进入运维中心-周期任务管理，点击上述任务操作列补数据按钮，点击确定 | 任务正常补数据失败 |
| 4 | 进入运维中心-补数据任务实例模块，点击上述任务名称查看实例详情-运行记录 | 日志框-提交日志模块正常新增时间戳，格式参考prd |
