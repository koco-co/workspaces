#### key范围校验主流程

##### 【P0】验证完整性校验-key范围校验（包含/Doris3.x）完整配置后规则集保存成功

> 前置条件

```
1、环境说明：数据质量模块已部署，通用配置-json格式配置中已维护以下key数据：
   根节点：key1（订单类型）、key2（商品名称）、key3（备注）
   key1的子节点：key11（普通订单）、key12（促销订单）

2、Doris3.x SQL语句准备:
DROP TABLE IF EXISTS qa_dq.test_json_table;
CREATE TABLE qa_dq.test_json_table (
    id INT,
    name VARCHAR(100),
    json_field STRING
) ENGINE=OLAP
DUPLICATE KEY(id)
DISTRIBUTED BY HASH(id) BUCKETS 1
PROPERTIES ("replication_allocation" = "tag.location.default: 1");
INSERT INTO qa_dq.test_json_table VALUES
(1, '记录A', '{"key1": "普通订单", "key2": "商品X"}'),
(2, '记录B', '{"key1": "促销订单"}'),
(3, '记录C', '{"key1": "普通订单", "key3": "无"}');
```
