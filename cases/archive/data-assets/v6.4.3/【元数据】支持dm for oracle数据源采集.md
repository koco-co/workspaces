## 数据资产-元数据

##### 验证「元模型管理」中存在DMDB for Oracle类型数据源图标且位置在Oracle右侧 「P3」

> 前置条件
```
无
```

| 编号 | 步骤 | 预期 |
| --- | --- | --- |
| 1 | 进入【元数据】-【元模型管理】页面 | 成功进入目标页面，页面内容正常加载显示，无报错 |
| 2 | 检查DM oracle数据源 | 针对DMDB for Oracle类型数据源的图标，类型放在Oracle的右侧 |
| 3 | 点击【编辑元模型】 | DMDB for Oracle默认采集的技术属性信息和Oracle类型保持一致 |

```
无
```

##### 验证「元数据同步」中DMDB for Oracle数据源下的库表周期同步功能正常 「P1」

> 前置条件
```
1) 数据资产-平台管理-数据源管理中已存在DMOracle数据源
2) DMOracle数据源已授权至当前质量项目
```

| 编号 | 步骤 | 预期 |
| --- | --- | --- |
| 1 | 进入【元数据】-【元数据同步】-【周期同步】页面 | 成功进入目标页面，页面内容正常加载显示，无报错 |
| 2 | 新增周期同步任务, 选择DMDB for Oracle类型的数据源下的库表进行同步 | 系统提示配置成功，配置项已生效，页面显示最新配置状态 |
| 3 | 点击立即同步 | 数据同步成功，系统给出成功反馈，相关页面/数据状态更新为最新 |

##### 验证「数据源管理」中可引用DMDB for Oracle类型的数据源 「P2」

> 前置条件
```
「公共管理-数据源中心」已存在DMOracle数据源
```

| 编号 | 步骤 | 预期 |
| --- | --- | --- |
| 1 | 进入【数据资产】-【平台管理】-【数据源管理】页面 | 成功进入目标页面，页面内容正常加载显示，无报错 |
| 2 | 点击【引入数据源】按钮 | 显示【数据源中心】中还未被资产平台引用的数据源列表 |
| 3 | 选择DMOracle数据源后, 点击确定 | 数据源成功引入至资产平台 |
