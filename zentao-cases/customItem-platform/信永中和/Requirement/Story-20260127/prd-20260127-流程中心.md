## 流程发起

#### **程序描述**

用户可在本页面选择流程类型，查看流程表单。填写、校验通过后，门户调用流程引擎发起流程。

流程的配置详见附录6-#2节点设计。

#### **需控制的菜单-按钮权限项**

| **一级模块** | **二级模块** | **三级页面** | **权限项**   | **页面/按钮** |
| ------------ | ------------ | ------------ | ------------ | ------------- |
| **流程中心** | **流程发起** | /            | 起点页       | 页面          |
|              |              | 数据标准管理 | 数据认责流程 | 页面          |




#### **功能详细设计**

本模块包含了以下功能：

![流程发起起点页](https://cdn.jsdelivr.net/gh/koco-co/picgo-images/picgo-images/%25E6%25B5%2581%25E7%25A8%258B%25E5%258F%2591%25E8%25B5%25B7%25E8%25B5%25B7%25E7%2582%25B9%25E9%25A1%25B5.png)

- 流程目录

按照附录6-#1-流程分类，显示完整二层流程目录。用户点击二级目录时校验角色权限。无权时显示‘您不是本流程的发起者’。



- 起点页: 如上图, 为默认页。
- 数据标准-数据认责流程. 注: 数据目录L3节点选择器，必选，可多选。只能选择L3，不可以选择L0-L2
  ![数据认责流程表单](https://cdn.jsdelivr.net/gh/koco-co/picgo-images/picgo-images/image-%25E6%2595%25B0%25E6%258D%25AE%25E8%25AE%25A4%25E8%25B4%25A3%25E6%25B5%2581%25E7%25A8%258B.png)

![发起流程表单](https://cdn.jsdelivr.net/gh/koco-co/picgo-images/picgo-images/image-20260226112638390.png)

需要注意的是: 只需要验证详情页中的数据显示正确就好了, 办理组件、边栏组件是调用的其它平台的接口, 不在此次的验证范围.



用户在本页面只选择L2。根据选择的L2内容，回显其他所有必填项。用户不可进一步维护。

填写内容详见附件6-#3-数据认责流程。

提交时需要做表单的完整性校验（可能回显null，提交时阻断）、或者重复性校验（当前未办结的同一流程中，不可有重复的L3）。

流程通过后，将附录1-#2-[新]数据目录架构（数据目录LO-L3主要信息）-是否已认证变为‘true’。

## 流程列表

#### **程序描述**

本模块调用云研平台-流程中心的公共组件搭建。用于在多个口径下查询流程列表信息。用户可以下载至详情页进行进一步查看或办理操作。

#### **需控制的菜单-按钮权限项**

| **一级模块** | **二级模块** | **三级页面** | **权限项**   | **页面/按钮** |
| ------------ | ------------ | ------------ | ------------ | ------------- |
| 流程中心     | 流程列表     | /            | 我的流程     | 页面          |
|              |              | /            | 我的待办     | 页面          |
|              |              | /            | 我的已办     | 页面          |
|              |              | /            | 我的待阅     | 页面          |
|              |              | /            | 我的转办     | 页面          |
|              |              | /            | 我的委托     | 页面          |
|              |              | /            | 我的受托     | 页面          |
|              |              | 流程详情     | 三级页面本体 | 页面          |



#### **功能详细设计**

本模块包含了以下功能：目录、列表、TAB

![流程列表详情页](https://cdn.jsdelivr.net/gh/koco-co/picgo-images/picgo-images/image-20260226105556330.png)



- 流程口径TAB-目录-列表

该部分嵌入云研平台-流程中心开放的流程列表。流程记录显示流程与云研平台-流程中心中的同一口径一致，***\*但是过滤掉门户流程目录（门户流程目录详见附录6-#1）之外\****的流程。流程标题（蓝字下划线）点击可进入流程详情页。

支持对流程标题（包含关键字）+流程目录（单选）进行搜索（and逻辑）。

支持点击TAB切换口径，口径包括我的流程、我的待办、我的已办、我的待阅、我的转办、我的委托、我的受托。



- 流程详情(与流程发起中的页面基本一致)

![流程详情页截图](https://cdn.jsdelivr.net/gh/koco-co/picgo-images/picgo-images/22a6a7c3-4697-45c6-8145-47ef0e6c975b.png)



流程详情页详情页包含三个部分：办理组件（原生集成）、边栏组件（原生集成）、详情本体（门户搭建）。

需要注意的是: 只需要验证详情页中的数据显示正确就好了, 办理组件、边栏组件是调用的其它平台的接口, 不在此次的用例编写范围.
