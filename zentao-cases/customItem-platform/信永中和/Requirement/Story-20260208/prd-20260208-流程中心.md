## 流程发起

#### **程序描述**

用户可在本页面选择流程类型，查看流程表单。填写、校验通过后，门户调用流程引擎发起流程。

流程的配置详见附录6-#2节点设计。

1、前端、后端、测试服务用的同一套环境, 不要分开. 单元测试、接口测试、自动化测试的代码放在/tests目录下的3个文件夹中, 方便查看管理.

2、引擎可以单独一个环境, 后续要打包到pypi中, 作为这个项目的核心依赖

3、Sisyphus-api-engine可以作为当前项目中的一个临时子项目, 需要根据Sisyphus-api-engine JSON 输出规范.md、Sisyphus-api-engine YAML 输入规范.md这两个文件编写详细的需求文档, 存放在引擎目录下的docs目录中, 另外把输入输出规范的两个文件也拷贝到docs目录中.

4、Sisyphus-api-engine子项目移动到项目根目录下. 这个依赖的开发规范也是按照当前项目的规范来. 这个子项目需要看作是一个单独的完整项目来进行开发. 子项目推荐的项目目录结构:

```bash
Sisyphus-api-engine/
├── .sisyphus/          # 统一环境配置
│   └── config.yaml
├── apirun/
│   ├── core/           # 核心数据模型
│   ├── parser/         # YAML 解析器
│   ├── executor/       # 测试执行器
│   ├── validation/     # 断言验证引擎
│   ├── extractor/      # 变量提取器
│   ├── data_driven/    # 数据驱动测试
│   ├── result/         # 结果收集器: json、html、allure、custom....
│   ├── websocket/      # WebSocket 实时推送
│   └── utils/          # 工具函数
└── ......              # .....
│   └── cli.py      
├── examples/           # 示例yaml用例
├── docs/               # 项目文档
└── tests/              # 单元测试
└── CHANGELOG.md
└── README.md
└── pypi_pulish.sh		# PYPI_API_TOKEN可以从系统变量中获取, 我已经设置好了: export PYPI_API_TOKEN=xxx
└── ......              # .....
```
