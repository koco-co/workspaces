## 数据质量

### 质量监控

#### 远程触发任务停止

##### 【125358】验证远程触发的质量监控任务可正常停止

> 前置条件
```
1、环境要求：测试环境，已部署 hotfix_5.3.x_125358 分支代码

2、数据准备：
  a. 已有一个可用的数据源（如 MySQL），且该数据源下存在至少一张表
  b. 已为该表创建一条单表质量监控规则，并已配置远程触发
  c. 获取该监控规则的远程触发 URL（在质量监控列表中通过"远程触发"功能生成）

3、账号要求：拥有数据质量-规则编辑权限的用户
```

> 用例步骤

| 编号 | 步骤 | 预期 |
| ---- | ---- | ---- |
| 1 | 使用有权限的账号登录系统 | 登录成功，进入首页 |
| 2 | 导航至【数据质量 > 质量监控】 | 页面正常加载，显示质量监控列表 |
| 3 | 找到已配置远程触发的监控规则，点击操作栏【远程触发】按钮 | 弹出远程触发配置弹框，显示触发 URL 和查看状态 URL |
| 4 | 复制远程触发 URL，通过 curl 或 Postman 发起 POST 请求触发该监控任务 | 请求返回成功，监控任务开始运行 |
| 5 | 返回质量监控列表，点击该监控规则进入监控详情/运行记录页面 | 显示一条状态为"运行中"的任务实例记录 |
| 6 | 在运行中的任务实例记录上，点击【停止】按钮 | 任务状态变为"停止中"，随后变为"已停止"，不再报错"实例未查询到" |
| 7 | 验证数据库中该条 monitor_record 记录的 flow_job_id 字段 | flow_job_id 字段值不为空，与 job_key 字段值一致 |
