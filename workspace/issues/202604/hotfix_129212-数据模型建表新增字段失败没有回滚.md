## 数据资产

### 数据模型 - 规范建表

#### 建表事务回滚

##### 【129212】验证规范建表新增字段失败后数据库记录完整回滚

> 前置条件

1. 环境：已部署包含 hotfix_6.0.x_129212 分支修复内容的后端服务（dt-center-metadata），column 表已完成分表配置。
2. 账号：使用具有「数据模型-规范建表」创建/编辑权限的账号登录数据资产平台。
3. 数据准备：在目标数仓层级下，已存在一个可用的 Hive 类型数据源及数据库（db），且该数据库中尚未存在同名数据表。
4. 网络/依赖：后端 Hive/数据源执行环境正常可用，以便能正常触发建表执行流程进入提交阶段。

> 用例步骤

| 编号 | 步骤 | 预期 |
| ---- | ---- | ---- |
| 1 | 登录数据资产平台，点击左侧导航「数据模型」，进入「规范建表」（标准化建表）页面 | 页面正常展示规范建表列表，顶部显示「新建表」按钮 |
| 2 | 点击「新建表」按钮，填写步骤一（基本信息）：输入合法的表名（英文）、表中文名，选择目标数仓层级、数据源、数据库，设置生命周期，点击「下一步」 | 跳转至步骤二（表结构配置）页面 |
| 3 | 在步骤二中，点击「新增字段」按钮，依次添加多个字段（至少 3 个），填写每个字段的字段名（英文）、字段中文名、字段类型；其中故意将某个字段名设置为当前数据源中已存在字段的保留关键字或超长字符串，使后续建表执行时能触发失败 | 字段行被添加到字段列表中，表单可正常填写 |
| 4 | 完成字段填写后，点击「提交建表」（或步骤三确认页的「建表」按钮），触发 POST /dataWarehouseTable/createOrUpdateTable 接口调用 | 接口请求发出，页面显示加载中状态 |
| 5 | 等待接口返回失败响应（因字段非法导致建表执行失败），观察页面提示信息 | 页面弹出明确的错误提示，说明建表失败原因；不显示「建表成功」或跳转至成功状态 |
| 6 | 回到「规范建表」列表页，搜索步骤二中填写的表名 | 列表中不存在该表记录，即建表失败后表记录已被完整回滚，未产生半成品记录 |
| 7 | 直接查询后端数据库（或通过运维接口），确认 data_table 及对应 data_table_column（分表）中均无该表及其字段的残留记录 | data_table 表和 data_table_column 分表中均无该次操作产生的数据行，事务回滚彻底 |
