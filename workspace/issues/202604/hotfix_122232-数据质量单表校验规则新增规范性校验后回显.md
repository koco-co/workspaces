## 数据质量

### 规则任务管理

#### 单表校验规则 - 规范性校验回显

##### 【122232】验证单表校验规则新增规范性校验后编辑页面可正确回显规则类型

> 前置条件

- 环境：assetv6.0 测试环境，已部署 hotfix_6.0.x_122232 修复版本
- 账号：具有数据质量模块操作权限的测试账号（DATAVALID_RULECONFIG 权限码）
- 数据准备：数据质量项目已存在，且项目内已配置可用数据源（如 Hive / MySQL 均可），数据源下存在可选数据表
- 前提：访问路径 数据质量 -> 规则任务管理（/dq/rule），页面可正常加载

> 用例步骤

| 编号 | 步骤 | 预期 |
| ---- | ---- | ---- |
| 1 | 进入数据质量模块，点击左侧菜单「规则任务管理」，进入规则任务管理列表页（/dq/rule） | 页面正常加载，顶部显示「新建监控规则」按钮 |
| 2 | 点击「新建监控规则」按钮，打开三步骤新建向导 | 向导弹出，当前处于第一步「基础信息」 |
| 3 | 第一步：填写规则名称（如：test_normative_122232），选择已配置的数据源，选择对应数据库，选择已有数据表，点击「下一步」 | 进入第二步「配置规则」 |
| 4 | 第二步：点击「新增规则」按钮，在规则类型下拉框中选择「规范性校验」 | 规则卡片标题区域显示「规范性校验」，卡片内展示规范性校验对应的配置项（如标准规则列表） |
| 5 | 在规范性校验规则卡片中，配置至少一个统计规则（如选择「字符串长度」统计函数，设置校验方法为「固定值」，设置期望值范围），点击「下一步」 | 进入第三步「调度配置」 |
| 6 | 第三步：保持默认调度配置，点击「保存」 | 提示保存成功，页面跳转回规则任务管理列表，列表中可见刚创建的规则 test_normative_122232 |
| 7 | 在规则列表中找到 test_normative_122232，点击操作列「编辑」按钮，进入编辑页面 | 跳转至编辑向导，自动进入第二步「配置规则」，或可手动切换至第二步 |
| 8 | 查看第二步「配置规则」中的规则卡片，确认卡片标题及内容 | 规则卡片标题显示为「规范性校验」（而非「自定义SQL」或其他类型），卡片内规范性校验专属配置项（标准规则列表、统计函数等）完整回显，之前配置的字符串长度规则及期望值配置均正确显示 |
