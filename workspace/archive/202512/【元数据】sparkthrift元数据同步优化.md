##### 【P1】验证编辑同步功能正确

> 前置条件

```
1.【元数据-元数据同步-周期同步】tab页已打开
2.列表已有sparkthrift数据源 test，该数据源下有多张数据库、数据表，某个数据库下有20W+ 张数据表，其中数据库test1-->数据表test11已进行了周期同步，调度周期为“天”；
3.数据库test2-->数据表test21已进行了临时同步
```

> 用例步骤

| 编号 | 步骤 | 预期 |
| ---- | ---- | ---- |
| 1 | 查看临时同步的数据源test-->数据库test2-->数据表test21 对应操作列 | 【编辑】按钮置灰 |
| 2 | 查看周期同步的数据源test-->数据库test1-->数据表test11 对应操作列 | 【编辑】按钮支持点击 |
| 3 | 点击【编辑】按钮 | 弹出【编辑周期同步任务】弹窗，数据回填正确； 【同步内容】页“选择数据”区域不支持更改 |
| 4 | 点击【临时同步】按钮 | 1）提示：临时同步成功； 2）列表新增该数据源的同步内容； 3）调度周期显示为“--”； 4）同步状态显示为“同步中”； 5）最近一次实例状态显示为“同步中” |
| 5 | 等待同步完成，点击该数据源查看同步内容 | 同步内容正确 |
| 6 | 重新点击该周期同步的数据源test的【编辑】按钮 | 弹出【编辑周期同步任务】弹窗，数据回填正确； 【同步内容】页“选择数据”区域不支持更改 |
| 7 | 点击【下一步】按钮 | 进入【调度配置】页面，数据回填正确 |
| 8 | 修改调度周期为“周”，选择时间为“星期一”，具体时间为“00时00分”，点击【保存并立即执行】 | 1）提示：编辑周期同步任务成功； 2）列表新增该数据源的同步内容； 3）调度周期显示为“周”； 4）同步状态显示为“同步中”； 5）最近一次实例状态显示为“同步中” |
| 9 | 等待同步完成，点击该数据源查看同步内容 | 同步内容正确 |
| 10 | 每周查看该数据源同步情况 | 每周一00时00分后自动同步，同步内容正确 |
