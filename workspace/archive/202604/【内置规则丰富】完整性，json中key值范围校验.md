> 前置条件

```

2) 已通过【数据质量 → 规则集管理】页面创建规则集"rule_set_key_range_test"，关联Doris数据源 qa_test 库 test_json_key_range 表（含字段 info（JSON类型）、extra_info（VARCHAR类型）），规则包名称"key范围校验测试包"
```

##### 【P1】验证校验方法切换（包含与不包含）规则保存和执行结果差异

> 前置条件

```
1) 已在【通用配置 → json格式校验管理】页面维护key数据：key1（姓名）、key2（年龄）

3) 已在Doris表test_json_key_range中插入以下数据：
   INSERT INTO test_json_key_range VALUES
     (1, '{"key1":"张三","key2":25}'),
     (2, '{"key1":"李四"}');
4) 已通过【数据质量 → 规则任务管理】页面，点击【新建监控规则】创建任务"task_json_method_switch"，关联同一Doris表，并在 Step 2 监控规则中通过【导入规则包】导入规则集管理中已配置的"method_switch包"，已完成 Step 3 调度属性并保存
```

| 编号 | 步骤 | 预期 |
| ---- | ------------ | ------------ |

| 7 | 进入【数据质量 → 校验结果查询】页面，找到"task_json_method_switch"最新实例记录并打开实例详情 | 1) id=1（含key1和key2，违反不包含规则）质检结果=校验不通过，详情说明=不符合规则key范围不包含"key1-key2"<br>2) id=2（仅含key1，部分包含）质检结果=校验不通过，详情说明=不符合规则key范围不包含"key1-key2" |
