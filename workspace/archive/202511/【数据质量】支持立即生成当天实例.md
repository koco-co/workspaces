##### 【P1】验证新增配置规则时，调度属性页面新增 [实例生成方式]

> 前置条件

```
无
```

> 用例步骤

| 编号 | 步骤 | 预期 |
| ---- | ---- | ---- |
| 1 | 进入数据质量-规则配置-调度属性页面, | 进入成功 |
| 2 | 查看页面 | 1.规则拼接包下方新增实例生成方式可选字段， |
| 3 | 查看字段右边“？”提示内容 | 支持选项T+1 & 立即生成； |
| 4 |  | 默认选择T+1 |
| 5 |  | 提示： 选中「T+1生成」，代表实例生成按照配置项设置的时间生成实例，默认是22：00； 选中「立即生成」，代表提交后立即生成当天开始未来时间内的实例，之后的实例生成还是按照配置项设置的时问生成实例； |

##### 【P0】验证新增规则时选中「T+1生成」时实例在配置时间生成

> 前置条件

```
1.数据质量-规则任务配置页面已打开
```

> 用例步骤

| 编号 | 步骤 | 预期 |
| ---- | ---- | ---- |
| 1 | 1.点击新增规则任务配置 | 配置完成 |
| 2 | 添加监控对象 | 进入成功 |
| 3 | 添加监控规则：完整性、准确性、规范性、唯一性校验 | 选择成功 |
| 4 | 点击下一步，进入调度属性页面 | 规则任务配置成功 |
| 5 | 选择实例生成方式为 T+1生成，其余必填属性内容正确填写完毕 | 1.成功运行 |
| 6 | 点击新建按钮 | 校验结果符合预期 |
| 7 | 运行任务,查看任务执行情况和质量报告 | 质量报告生成成功,内容正确 |
| 8 | 查看该规则生成实例状态 | 未立即生成 |
| 9 | 第二天22点前查看实例生成状态 | 未生成 |
| 10 | 第二天22点后查看实例生成状态 | 成功生成前一天配置的规则任务实例，显示生成时间为22点 |
