# Hotfix 用例 Frontmatter 偏好

## keywords 字段

Hotfix 用例的 frontmatter 中**必须**包含 `keywords` 字段，用于快速分类检索。

格式: `"【1】发现的大版本 | 【2】模块 | 【3】数据源类型 | 【4】集群 | 【5】真实解决的最低版本 | 【6】问题原因"`

各字段说明:

| 位置 | 含义 | 示例值 | 备注 |
| ---- | ---- | ------ | ---- |
| 1 | 发现的大版本 | `6.3` | 首次发现该 Bug 的产品大版本 |
| 2 | 模块 | `数据模型`、`数据质量`、`元数据` | 所属功能模块 |
| 3 | 数据源类型 | `MySQL`、`Hive`、`Doris` | 若与数据源无关则留空 |
| 4 | 集群 | `standalone`、`HA` | 若与集群无关则留空 |
| 5 | 真实解决的最低版本 | `6.3` | 实际合入修复的最低版本 |
| 6 | 问题原因 | `需求变更`、`代码缺陷`、`配置错误` | 根因分类 |

示例:

```yaml
---
suite_name: "Hotfix 用例 - xxx"
keywords: "6.3 | 数据模型 | | | 6.3 | 需求变更"
tags:
  - hotfix
---
```

留空字段保留分隔符 `|`，确保位置对齐。
