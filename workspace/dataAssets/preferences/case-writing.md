## 1. 数据质量模块 — 菜单结构与导航路径

数据质量（`/dq`）侧边栏菜单（按显示顺序）：

| 菜单名称     | 路由                | 说明                                            |
| ------------ | ------------------- | ----------------------------------------------- |
| 总览         | `/dq/overview`      | 数据质量总览仪表盘                              |
| 规则库配置   | `/dq/ruleBase`      | 内置规则、自定义规则、SQL 模板的管理            |
| 规则集管理   | `/dq/ruleSet`       | 按"数据表"维度组织规则包，供规则任务引用        |
| 规则任务管理 | `/dq/rule`          | 创建/编辑单表监控规则任务，可从规则集导入规则包 |
| 校验结果查询 | `/dq/taskQuery`     | 查看校验执行结果                                |
| 数据质量报告 | `/dq/qualityReport` | 质量报告查看与配置                              |
| 通用配置     | `/dq/generalConfig` | 报告关联维表设置等                              |
| 项目管理     | `/dq/project`       | 项目信息、脏数据管理                            |

**核心业务关系**：规则库配置 → 规则集管理 → 规则任务管理

1. 先在「规则库配置」中管理内置/自定义规则模板
2. 再在「规则集管理」中按数据表维度组织规则包
3. 最后在「规则任务管理」中创建监控任务，通过"导入规则包"引用规则集

> 用例中**不得**跳过规则集直接在规则任务中创建质量规则。规则任务通过"导入规则包"引用规则集中已配置好的规则。

## 2. 规则集管理 — 表单结构

### 2.1 列表页

- 导航：数据质量 → 规则集管理
- 主按钮：【新建规则集】
- 列表列：表名、所属数据库、所属数据源、规则包数量、规则数量、规则集描述、更新人、更新时间、操作
- 行操作：编辑、删除
- 搜索：输入表名搜索
- 删除提示：「删除规则集不会影响已经建好的规则任务」

### 2.2 新建/编辑规则集（两步表单）

**Step 1：基础信息**

表单字段（视觉顺序）：

- \*选择数据源（Select，编辑时禁用）
- \*选择数据库（Select，编辑时禁用）
- \*选择数据表（Select，编辑时禁用）
- 规则集描述（Input，最长 255 字符）
- \*规则包名称（Table 形式，支持新增/删除，最多 20 个，最长 50 字符，不可重复）

底部按钮：【取消】【下一步】

**Step 2：监控规则**

- 按规则包分组配置具体的监控规则
- 底部按钮：【上一步】【保存】
- 支持「新增规则包」和「查看全局参数」

> 数据源类型以需求实际实现为准；当前项目常见为 Hive2.x、SparkThrift2.x、Doris 系列（版本按需求来源 / 源码确认，如 Doris2.x 或 Doris3.x）。

## 3. 规则任务管理 — 表单结构

### 3.1 列表页

- 导航：数据质量 → 规则任务管理
- 主按钮：【新建监控规则】
- 列表列：表、任务名称、数据源、执行周期、规则状态、是否关联任务、最近30天告警数、最近修改人、最近修改时间、操作
- 批量操作：开启检测、关闭检测
- 搜索：输入表名搜索、最近修改人筛选、我收藏的表

### 3.2 新建/编辑单表监控规则（三步表单）

**Step 1：监控对象**

表单字段（视觉顺序）：

- \*规则名称（Input）
- \*选择数据源（Select）
- \*选择数据库（Select，编辑时禁用）
- \*选择数据表（Select，编辑时禁用）
- 选择分区 / 输入分区
- 全局参数
- 抽样检查设置

底部按钮：【取消】【下一步】

**Step 2：监控规则**

- 添加内置规则或自定义 SQL 规则
- 可通过「导入规则包」从已有规则集中引入规则
- 底部按钮：【上一步】【下一步】【临时保存】

**Step 3：调度属性**

- 调度配置（执行周期、起止时间等）
- 规则拼接包
- 资源组
- 告警配置
- 任务关联
- 报告配置
- 实例生成方式（T+1 生成 / 立即生成）
- 环境参数配置

底部按钮：【上一步】【保存】【临时保存】

## 4. 规则库配置 — 结构

### 4.1 页面 Tab

- 导航：数据质量 → 规则库配置
- 三个 Tab：内置规则、自定义规则、SQL 模板

### 4.2 内置规则

- 列表列：函数名称、函数说明、规则任务类型、关联范围、关联数量、启用状态（Switch）、描述
- 支持导出

### 4.3 自定义规则

- 支持新增、编辑、删除、查看详情

### 4.4 SQL 模板

- 支持新增、编辑、删除、查看详情

## 5. 按钮名称规范

用例中引用的按钮名称必须与系统实际 UI 一致。以下为常见易错对照：

| 正确名称           | 常见错误写法                 |
| ------------------ | ---------------------------- |
| 【新建规则集】     | 【新增规则】【创建规则集】   |
| 【新建监控规则】   | 【新建规则任务】【添加规则】 |
| 【下一步】         | 【下一步(2/3)】【继续】      |
| 【保存】           | 【确定】【提交】             |
| 【取消】           | 【返回】【关闭】             |
| 【导入规则包】     | 【引用规则】【添加规则包】   |
| 【新增】(规则包行) | 【添加规则包】               |

## 6. 需求与数据源类型对应关系

不同需求来源使用的主要数据源类型不同，编写用例时需结合需求来源与源码 / PRD 中确认的实际版本选择对应数据源：

| 需求来源 | 主要数据源类型 |
| -------- | -------------- |
| 岚图需求 | SparkThrift2.x                         |
| Gate需求 | Doris 系列（版本以需求实际实现为准） |

> 用例中建表语句、字段类型、SQL 语法需与对应数据源兼容（如 SparkThrift 使用 `STRING`，Doris 系列按实际版本使用对应语法，如 `VARCHAR` 等）。
