##### 【P2】验证离线建表设置tableLife生效逻辑正确(53版本及以上)

> 前置条件

```
CREATE TABLE user_log (
uid BIGINT,
action STRING,
ts STRING
)
PARTITIONED BY (dt STRING)
tableLife 1;
```

> 用例步骤

| 编号 | 步骤 | 预期 |
| ---- | ---- | ---- |
| 1 | 进入离线开发平台，创建${DATASOURCE}类型SQL任务 | 创建成功 |
| 2 | 输入前提条件内的建表语句，保存 | SQL任务保存成功 |
| 3 | 运行当前任务 | 任务运行成功，表创建成功 |
| 4 | 1天后，再次查看表是否存在 | 表生命周期到期，表被删除 |
| 5 | 离线创建表A时，设置生命周期为10， 同步到资产后，资产维护表生命周期为1 | 资产维护表A生命周期成功 |
| 6 | 离线再次执行建表SQL，设置生命周期为10，执行SQL | 执行成功 |
| 7 | 1天后，查看表A | 表A仍被删除(离线建表逻辑为create table if not exists),对于已存在并设置生命周期的表不会再生效 |
| 8 | 查看审计日志 | 审计日志记录正确 |
