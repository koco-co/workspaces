### 规则集管理

#### 规则配置-新增取值范围&枚举范围规则（P0核心流程）

##### 【P0】验证在规则集中配置取值范围&枚举范围规则且关系完整流程（数值类型字段）

> 前置条件
```
1) 使用 admin 账号登录数据资产平台，已进入当前 quality 项目（pw_test 租户）
2) 已在 SparkThrift2.x 数据源 pw_test 库执行以下建表和灌数 SQL：
   DROP TABLE IF EXISTS pw_test.quality_test_num;
   CREATE TABLE pw_test.quality_test_num (
     id INT,
     score DOUBLE,
     category STRING
   ) STORED AS PARQUET;
   INSERT INTO TABLE pw_test.quality_test_num
   SELECT 1, 5.0, '2'
   UNION ALL
   SELECT 2, 15.0, '4'
   UNION ALL
   SELECT 3, 3.0, '1'
   UNION ALL
   SELECT 4, -1.0, '3'
   UNION ALL
   SELECT 5, 8.0, '5';
   -- 数据语义说明：
   -- score 字段期望 >1 且 <10；category 枚举值期望 in '1,2,3'
   -- 满足且关系：id=1(score=5,category=2)、id=3(score=3,category=1)
   -- 不满足：id=2(score=15 越界)、id=4(score=-1 越界)、id=5(category=5 不在枚举)
3) 已通过【数据质量 → 规则集管理】页面，点击【新建规则集】，在 Step 1 基础信息中选择数据源=SparkThrift2.x、数据库=pw_test、数据表=quality_test_num，规则包名称="且关系校验包"，点击【下一步】进入 Step 2 监控规则后直接点击页面底部【保存】完成规则集 "ruleset_15695_and" 创建（规则包内暂无规则）
```

| 编号 | 步骤 | 预期 |

| 2 | 在搜索框输入 "ruleset_15695_and" 定位规则集，点击操作列的【编辑】按钮，等待 Step 1 基础信息页加载，点击【下一步】进入 Step 2 监控规则页面 | Step 2 监控规则页面正常打开，已有规则包 "且关系校验包" 回显在页面中 |
| 3 | 在规则包 "且关系校验包" 行右侧点击【添加规则】下拉按钮，在弹出的规则类型下拉菜单中选择【有效性校验】 | 规则包下方新增一行规则配置区域，显示统计函数下拉框、字段下拉框、强弱规则下拉框、规则描述输入框 |
| 4 | 在新增规则行的统计函数下拉框中选择【取值范围&枚举范围】 | 规则配置区域展开为 3 行布局：1) 「取值范围设置：」行（含期望值输入框 + 操作符下拉框 + 右侧 +/- 按钮）；2) 「枚举值设置：」行（含 in/not in 下拉框 + 枚举值输入框）；3) 「取值范围和枚举值的关系：」行（且/或单选按钮 + 过滤条件） |

| 6 | 点击页面底部【保存】完成规则集保存 | 1) 弹出保存成功的提示「保存成功」；2) 自动返回规则集管理列表页 |
| 7 | 在列表中找到 "ruleset_15695_and"，点击操作列的【编辑】按钮，进入 Step 2 监控规则页面，查看新增规则的回显 | 规则包 "且关系校验包" 中已新增一条规则，回显内容如下：<br>1) 字段：score<br>2) 取值范围设置：>1（且）<10（取值范围内的且或单选按钮回显为「且」）<br>3) 枚举值设置：in 1,2,3<br>4) 取值范围和枚举值的关系：「且」单选按钮被选中<br>5) 强弱规则：强规则<br>6) 规则描述：score 取值范围 1 到 10 且枚举值 in 1,2,3 |

使用 Doris3.x 数据源重复以上步骤，将 `score` 字段类型保持 `DOUBLE`、`category` 字段类型从 `STRING` 改为 `VARCHAR(50)`，建表语句改为 Doris3.x DDL 模板（`DISTRIBUTED BY HASH(id) BUCKETS 3 PROPERTIES("replication_num"="1")`），其余 UI 操作不变；预期结果与 SparkThrift2.x 一致。
