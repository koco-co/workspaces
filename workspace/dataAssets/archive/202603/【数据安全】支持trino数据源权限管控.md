##### 【P0】验证【开发用户】-【资产-元数据-数据地图-数据表】-【权限申请】-【trino数据源】配置功能正确

> 前置条件

```
1、平台存在trino数据源：trino_test
2、trino数据源内存在catalog：catalog1、catalog2
3、该数据源-catalog1下存在数据库：schema1
4、schema1下存在数据表table1
5、管理员已进入【资产-数据安全-数据权限管理-新增权限策略】页面
6、平台已存在身份为数据开发的用户：ty
```

> 用例步骤

| 编号 | 步骤 | 预期 |
| ---- | ---- | ---- |
| 1 | 登录ty用户账密，进入【资产-元数据-数据地图】页面 | 页面正常打开 |
| 2 | 选择trino数据表：table1，点击【批量申请权限】 | 抽屉式展开表权限申请页 |
| 3 | 鼠标悬浮【表级权限】处的“？” | 提示：“hive、sparkthrift、starrocks、doris、trino、inceptor类型数据源配置权限信息后可在离线开发平台生效，其他类型数据源配置权限信息后仅在资产平台生效” |

| 5 | 【权限】勾选DQL，【行列权限配置】开启行列级，配置【行列权限】为schema1-table1-字段1&字段2；【有效期】默认为永久，【申请原因】输入test，点击【确定】 | 配置成功 |
| 6 | 打开【资产-元数据-数据地图-数据表table1】，查看数据预览 | 1）支持查看表table1的字段 2）只支持显示字段1和字段2的数据 |
| 7 | 打开【数据质量】，新建规则任务，选择trino_test-catalog1-schema1-table1，查看数据预览 | 1）支持数据预览 2）只有字段1和字段2的数据显示 |
| 8 | 切换至【离线开发】，查询trino表table1 | 支持查询，结果为行列级：table1：字段1和字段2 |
