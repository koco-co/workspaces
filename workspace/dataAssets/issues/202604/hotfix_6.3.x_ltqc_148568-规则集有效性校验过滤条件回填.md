## 数据质量

### 规则集管理

#### 有效性校验规则 - 过滤条件配置

##### 【148568】验证有效性校验规则配置过滤条件保存后可正常回填显示

> 前置条件

```
1、环境要求：岚图生产对应的测试环境，已部署 hotfix_6.3.x_ltqc_148568 修复版本

2、数据准备：
   确保已存在可用的规则集，或在测试中新建规则集。
   规则集中需包含「有效性」类型校验规则，且规则类型选择「数据取值范围」（对应后端 FunctionType.VALUE_RANGE）。

3、账号要求：具有数据质量模块操作权限的账号（可创建/编辑规则集）
```

> 用例步骤

| 编号 | 步骤 | 预期 |
| ---- | ---- | ---- |
| 1 | 使用有数据质量权限的账号登录系统 | 登录成功，进入系统首页 |
| 2 | 导航至【数据质量 > 规则集管理】 | 页面正常加载，显示规则集列表 |
| 3 | 点击「新建规则集」或打开已有规则集进入编辑页面 | 规则集编辑页面正常打开 |
| 4 | 切换到「有效性」校验规则选项卡，点击「添加规则」，规则类型选择「数据取值范围」 | 规则配置表单展开，出现过滤条件配置区域 |
| 5 | 在过滤条件输入框中填写过滤条件，例如：`status = 1` | 过滤条件输入框正常显示输入内容 |
| 6 | 完整填写规则其他必填项（字段名、取值范围等），点击「保存」或「确认」按钮提交规则集 | 规则集保存成功，无报错提示 |
| 7 | 关闭当前规则集编辑页面，重新从规则集列表点击进入刚才保存的规则集 | 规则集详情/编辑页面正常打开 |
| 8 | 切换到「有效性」校验规则选项卡，找到刚才配置的「数据取值范围」规则，查看过滤条件字段 | 过滤条件字段显示之前输入的内容（如 `status = 1`），不为空 |
| 9 | 再次编辑该规则，修改过滤条件为其他值（如 `status = 2`），保存后重新进入页面 | 修改后的过滤条件正确回填，与修改值一致，不被覆盖为空 |
