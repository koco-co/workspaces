## 数据模型

### 规范建表 > 建表

#### 字段中文名解析时的词根翻译逻辑

##### 【148219】验证字段中文名解析仅触发整词精确匹配词根，不再进行分词拼接匹配

> 前置条件

```
无
```

> 用例步骤

| 编号 | 步骤 | 预期 |
| ---- | ---- | ---- |
| 1 | 进入【数据标准 > 标准基础 > 词根管理】 | 词根管理页面正常加载 |
| 2 | 新建词根:<br>- *词根简称: Sales_dept<br>- 词根全称: Sales_dept<br>- 词根中文名: 采销部 | 新建成功，列表中可查到该词根 |
| 3 | 进入【数据模型 > 规范建表 > 建表】 | 建表列表页正常加载 |
| 4 | 点击新建表，填写基础信息:<br>- *写入数据源: SparkThrift2.x<br>- *写入数据库: test_db<br>- 数仓层级: ODS<br>- *表名: test_root_148219<br>- *表中文名: 词根验证表<br>- *存储格式: orc<br>- *生命周期: 永久<br>- *表类型: 内部表<br>- hdfs存储路径:<br><br>点击下一步 | 进入表结构编辑页面 |
| 5 | 选择批量解析模式，输入:<br><br>采销部<br><br>采销部部长<br><br>点击解析 | 解析成功，生成两条字段记录:<br>- "采销部"精确匹配词根，字段英文名填充为 Sales_dept<br>- "采销部部长"无整词匹配，字段英文名为空（不出现 Sales_dept 等分词拼接结果），系统不报错 |
