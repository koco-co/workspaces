## 数据质量

### 规则任务管理

#### 多表校验规则配置

##### 【123506】验证多表校验在 KingBase 数据源下远程搜索选择表后不再触发额外的数据表列表请求

> 前置条件

```
1、环境要求：已部署 5.2.x 版本，含 dataAssets/hotfix_5.2.x_123506 分支修复；
   系统中已配置并连通 KingBase 数据源，该数据源下有可查询的数据表（不少于 5 张）。

2、数据准备：
   （1）确认 KingBase 数据源已在 控制台 > 数据源管理 中注册，状态为"连接成功"；
   （2）在 数据质量 > 项目管理 > 项目信息 中已创建至少一个数据质量项目，且该项目关联上述 KingBase 数据源；
   （3）打开浏览器开发者工具（F12），切换到 Network 面板，准备记录接口请求。

3、账号要求：具备数据质量模块【规则任务管理】权限的账号（DATAVALID_RULECONFIG）。
```

> 用例步骤

| 编号 | 步骤 | 预期 |
| ---- | ---- | ---- |
| 1 | 使用有权限账号登录数据资产系统 | 登录成功，进入系统首页 |
| 2 | 顶部导航切换至【数据质量】模块，点击左侧菜单【规则任务管理】 | 页面正常加载，显示规则任务列表 |
| 3 | 点击页面右上角【新建规则】按钮 | 弹出规则创建向导，进入第一步【监控对象】 |
| 4 | 在【监控对象】步骤中，选择步骤一数据源类型为 KingBase，选择已配置的 KingBase 数据源、Catalog（如有）和数据库 | 数据源选择成功，规则类型列表加载完成 |
| 5 | 在规则类型处选择多表校验规则（如"多表唯一性判断"或"多表内容对比"），页面展示多表配置区域，包含数据表搜索下拉框 | 多表配置区域正常展示，可见【选择数据表】下拉框，Network 面板中出现一次获取数据表列表的请求（初始加载前 200 条） |
| 6 | 在 Network 面板中清空已有请求记录（点击清除按钮），然后在【选择数据表】下拉框中输入关键字（如表名前几个字符）进行远程搜索 | 下拉框触发一次带 `search` 参数的数据表搜索接口请求，返回匹配的表列表；Network 中仅出现一次含 search 参数的请求，无报错 |
| 7 | 在远程搜索结果列表中，单击选中其中一张表 | 下拉框显示所选表名，选择操作成功；**Network 面板中不再出现新的数据表列表请求（即不触发额外的不含 search 参数的全量数据表请求）；页面无错误弹窗，KingBase 不报错** |
| 8 | 观察页面是否出现错误提示（Toast/Modal 弹窗） | 页面无任何错误提示，规则配置表单状态正常，可继续进行后续步骤 |
| 9 | 点击【下一步】完成步骤一配置，验证规则配置流程可正常进入步骤二【监控规则】 | 成功进入步骤二，表明多表校验的表选择逻辑运行正常 |
